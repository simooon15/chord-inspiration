## 变更背景

V2.1 解决了节奏型和真实吉他音色的问题。但一个痛点仍然存在：

**数据困在浏览器里。** V2.0/V2.1 的收藏、笔记、录音全存在 LocalStorage 和 IndexedDB 里，清缓存就没了，换设备也带不走。用户积累了几十组素材后开始焦虑——这些心血没有备份，也不敢清理浏览器数据。

**决策：做导出/导入系统。** 多选打包下载 .zip，含和弦数据、笔记和录音，可在浏览器间迁移。在 V2.0 工程基座上增量构建，不改变技术架构。

> 注：钢弦吉他音色已在 V2.1 中实现（spessasynth_lib + SoundFont 真人采样），V2.2 不再重复。V2.2 可在已有基础上增加合成器/吉他音色切换开关（简单 toggle）。

## 1. 产品概述

### 1.1 一句话定位

给创作者加上数据备份能力（收藏多选导出/导入 .zip）和钢弦吉他音色，让素材不再困在浏览器里，试听更真实。

### 1.2 谁需要它

与 V2.0 一致，目标用户未变。核心场景：

- 用户积累了几十组收藏和录音后，想备份到硬盘或迁移到新设备/新浏览器
- 用户在试听和弦进行时，想用接近吉他原声的音色来更好地判断编曲效果
- 用户在不同设备间切换使用（如公司电脑 → 家里笔记本），希望数据跟随

### 1.3 核心价值

收藏的多选打包下载，含和弦数据、笔记和录音，可在浏览器间迁移。导入时冲突预览，用户决定覆盖/跳过/保留。

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 导出使用率 | ≥10% | 收藏列表中有导出操作的用户占比 |
| 导入使用率 | ≥5% | 有导入操作的用户占比 |

### 1.5 不做什么

- 不做云端同步（属于 V4.0 后端）
- 不做增量同步
- 不做录音格式转换、重新编码
- 不做按拍切换和弦（后续版本）

## 2. 功能范围

### 2.1 In / Out

| 维度 | V2.2 做 | V2.2 不做 |
|------|---------|-----------|
| 导出 | 收藏多选打包 zip，含和弦数据+笔记+录音 | 云端同步、增量同步 |

| 导入 | 选 zip → 预览 → 冲突处理 → 写入存储 | 自动合并、增量导入 |

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P1 | 导出收藏 | 管理模式多选 + jsZip 打包下载，含录音 |
| P1 | 导入收藏 | 选 zip → 预览 → 冲突处理 → 写入存储 |
| P2 | 钢弦吉他音色 | Karplus-Strong 合成，播放栏切换按钮 |

## 3. 功能详情

### 3.1 导出收藏（P1）

#### 3.1.1 导出流程

- **入口**：收藏列表（创作工作台）工具栏的「导出」按钮
- **管理模式**：点击导出按钮后，收藏列表进入管理模式，每个进行左侧显示复选框，顶部显示「全选/取消」
- **选择与打包**：勾选后点击「导出选中」按钮 → 自动生成默认文件名 `chord-inspiration-YYYY-MM-DD.zip`（用户可修改）→ 打包 zip → 触发浏览器下载
- **打包内容**：每个选中进行一个文件夹，文件夹名格式为 `1-5-6-4 进行` 或 `1-4-3-6-2-5-1 进行（卡农进行）`
- **每个文件夹内**：
  - `chords.json` — 机器可读的完整数据进行备份，供导入恢复
  - `chords.md` — 人类可读的 Markdown 格式，包含进行名称、调性、和弦表、示例歌曲
  - `recordings/` — 录音文件，命名沿用录音 ID

#### 3.1.2 zip 文件结构

```
chord-inspiration-2026-05-23.zip
├── export-v1/
│   ├── metadata.json                   ← 导出元信息（日期、版本号等）
│   ├── 1-5-6-4 进行（卡农进行）/
│   │   ├── chords.json
│   │   ├── chords.md
│   │   └── recordings/
│   │       ├── memo-uuid-1.webm
│   │       └── memo-uuid-2.webm
│   └── 6-4-1-5 进行/
│       ├── chords.json
│       ├── chords.md
│       └── recordings/
```

### 3.2 导入收藏（P1）

#### 3.2.1 导入流程

- **入口**：收藏列表工具栏的「导入」按钮
- **文件选择**：弹出 `<input type="file" accept=".zip">`，仅接受 zip 文件
- **预览**：选择 zip 后解析，弹出预览弹窗，列出 zip 内所有进行
  - 每项标注该进行是否已在本地收藏中存在（冲突检测依据：progression ID 匹配）
  - 有冲突的进行提供三个操作选项：**覆盖**（用导入的笔记/录音替换本地）、**跳过**（保留本地，不导入该项）、**保留两份**（导入的数据作为新项加入收藏）
- **执行导入**：用户确认后，逐项写入 localStorage（收藏列表 + 笔记）和 IndexedDB（录音 Blob）
- **完成提示**：显示摘要（如「成功导入 3 个进行，跳过 1 个，保留两份 0 个」）

#### 3.2.2 chords.json 格式

```json
{
  "version": 1,
  "progression": {
    "id": "pop_classic_heart",
    "displayName": "1-5-6-4 进行（卡农进行）",
    "style": "pop",
    "chords": [
      { "rootOffset": 0, "quality": "Major", "nashvilleDegree": "1" },
      { "rootOffset": 4, "quality": "Major", "nashvilleDegree": "5" }
    ],
    "description": "经典流行曲走向，温暖而富有情感",
    "exampleSongs": ["《晴天》", "《Lemon》"]
  },
  "notes": "用户的笔记文本",
  "recordings": [
    { "id": "memo-uuid-1", "title": "试录片段", "date": "2026-05-20", "duration": "0:32" }
  ]
}
```

#### 3.2.3 chords.md 格式

```markdown
# 1-5-6-4 进行

**调性：** C
**风格：** 流行
**描述：** 经典流行曲走向，温暖而富有情感

## 和弦

| 序号 | 和弦 | 级数（罗马） | 级数（纳什维尔） | 描述 |
|------|------|-------------|----------------|------|
| 1 | C | I | 1 | 主和弦 |
| 2 | G | V | 5 | 通往下一级 |
| 3 | Am | vi | 6- | 离调到关系小调 |
| 4 | F | IV | 4 | 下行到结尾 |

## 示例歌曲

- 《晴天》
- 《Lemon》
```

### 3.3 钢弦吉他音色（P2）

- **切换方式**：播放栏「循环播放」按钮和「记谱」切换之间新增双态点击按钮，显示 `电子 / 吉他`，点击即切换，选中态高亮
- **音色实现**：使用 Web Audio 的 **Karplus-Strong 拨弦合成算法**——噪声脉冲触发 → 延迟线 + 低通滤波反馈，模拟钢弦的物理振动过程
- **电子音色**：沿用 V2.0/V2.1 的三角波 + 低通逻辑，不做改动
- **切换效果**：切换后即时生效，不影响正在播放的声音，下次播放时应用新音色
- **兼容**：节奏型 ON/OFF 两种播放模式均受音色切换影响

### 3.4 与 V2.1 的关系

- V2.2 在 V2.1 的基础上增量构建
- 导出/导入不依赖节奏型系统的任何代码，可独立开发
- 钢弦吉他音色依赖 V2.1 的 `strumChord()` 方向感知扫弦接口，需要在 V2.1 完成后开发
- 如果用户跳过了 V2.1 直接上 V2.2（技术上可行但在 V2.1 先发布的前提下不会发生），吉他音色仍兼容 V2.0 的 `playChord()` 接口

## 4. 验收标准

### US-1 (P1)：导出收藏

**As a** 收藏了多组和弦进行的用户
**I want to** 从收藏列表中选中一些进行并导出为 .zip 文件
**so that** 在另一台设备上恢复或备份数据

- **Scenario 管理模式选择**
  - Given 收藏列表中有 3 个进行
  - When 点击「导出」按钮
  - Then 列表进入管理模式，每项出现复选框，顶部出现全选/取消按钮

- **Scenario 打包下载**
  - Given 勾选了 2 个进行
  - When 点击「导出选中」按钮
  - Then 浏览器下载 `chord-inspiration-YYYY-MM-DD.zip`，解压后每个进行独立文件夹，内含 chords.json、chords.md、recordings/

### US-2 (P1)：导入收藏

**As a** 导出过收藏想在新浏览器恢复的用户
**I want to** 选择 .zip 文件后看到预览、确认后导入
**so that** 数据完整恢复到本地

- **Scenario 导入预览**
  - Given 导出的 .zip 包含 2 个进行，其中 1 个已在当前收藏中
  - When 选择该 zip 并上传
  - Then 预览弹窗显示 2 个进行，标注哪个有冲突，每项提供覆盖/跳过/保留两份选项

- **Scenario 执行导入**
  - Given 预览确认后
  - When 用户点击确认导入
  - Then 收藏列表更新、笔记恢复、录音恢复到 IndexedDB，显示完成摘要

### US-3 (P1)：导出导入含录音

**As a** 录了和弦试听的用户
**I want to** 导出的 zip 中包含录音文件，导入后录音可正常播放
**so that** 录音数据不丢失

- **Scenario 导出含录音**
  - Given 某个进行有 2 段录音
  - When 导出该进行
  - Then zip 内 `recordings/` 目录包含对应录音文件

- **Scenario 导入恢复录音**
  - Given 导入含录音的 zip
  - When 导入完成后打开该进行的录音列表
  - Then 录音列表显示导入的录音，点击可正常播放

### US-4 (P2)：切换音色

**As a** 想听到更真实吉他音色的用户
**I want to** 在播放栏切换电子音色和钢弦吉他音色
**so that** 试听时有更真实的音色选择

- **Scenario 切换至吉他**
  - Given 音色切换按钮显示「电子」
  - When 点击切换按钮
  - Then 按钮显示变为「吉他」，下次播放时使用钢弦吉他合成音色

- **Scenario 切换回电子**
  - Given 音色切换按钮显示「吉他」
  - When 点击切换按钮
  - Then 按钮显示变回「电子」，下次播放时恢复三角波低通音色

## 5. 技术设计

### 5.1 导出/导入模块

| 模块 | 职责 | 新增/修改 | 依赖 |
|------|------|-----------|------|
| `FavExport` | 管理模式 UI（进入选择态、复选框、全选/取消）、导出/导入按钮入口 | 新增 | React state |
| `ZipPacker` | 接受选中进行列表 → 从 localStorage 拉笔记 → 从 IndexedDB 拉录音 Blob → 用 jsZip 组装 zip → 触发下载 | 新增 | jsZip, IndexedDB |
| `ZipParser` | 接受用户选择的 zip → 解析 → 提取进行列表 + 冲突检测 → 返回预览数据 | 新增 | jsZip |
| `ImportEngine` | 接受用户的导入确认 → 写回 localStorage（收藏+笔记）→ 写回 IndexedDB（录音 Blob）→ 返回完成摘要 | 新增 | IndexedDB |

模块依赖关系：

```
用户操作
  ├──「导出」→ FavExport(管理模式) → ZipPacker(打包) → 下载
  └──「导入」→ 文件选择 → ZipParser(解析+冲突检测) → 预览确认 → ImportEngine(写入) → 完成提示
```

**数据流：**
- **导出**：`localStorage.getItem('and_xian_favorite_progressions')` → 匹配选中项 → `localStorage.getItem('and_xian_progression_notes')` 获取笔记 → `getAudioMemo(id)` 从 IndexedDB 逐条拉取录音 Blob → jsZip 组装 → `URL.createObjectURL` + `<a download>` 触发下载
- **导入**：`<input type="file">` 获取 File → jsZip 解析 → 预览确认 → `setFavorites()` 更新收藏 → 写回 localStorage 笔记 → `saveAudioMemo(id, blob)` 逐条写回 IndexedDB

### 5.2 和弦块吉他音色

> 节奏型系统中的扫弦吉他音色已在 V2.1 实现（三角波拨弦合成）。V2.2 在保留 V2.1 扫弦音色的基础上，为**和弦块播放**（点击和弦卡片 / 非节奏模式）增加完整钢弦吉他音色。

| 模块 | 职责 | 新增/修改 | 依赖 |
|------|------|-----------|------|
| `audio.ts` — `AudioSynth.setTone()` | 接受 `'synth'` / `'guitar'` 参数，切换 `playChord()` / `strumChord()` 内部的合成分支 | 修改 | AudioContext |
| `App.tsx` — `toneMode state` | 管理当前音色状态，传递给播放栏 UI 和 synth 实例 | 修改 | React state |

**Karplus-Strong 实现概要：** 每个吉他和弦音分解为独立拨弦事件，使用 Web Audio 的延迟线（`DelayNode` + `GainNode` 反馈回路）配合低通滤波模拟钢弦振动。每个音高由延迟时间决定（`sampleRate / frequency`），反馈系数控制衰减速度。

**音色切换不影响播放状态：** 切换只改 `synth.toneMode` 标志位，不中断正在播放的声音。下次 `playChord()` / `strumChord()` 调用时按新模式合成。

## 6. 与 V2.1 的差异总结

| 维度 | V2.1 | V2.2 |
|------|------|------|
| 数据备份 | 无 | 收藏多选导出/导入 zip，含笔记和录音 |
| 音色 | 电子音色（和弦块）+ 三角波拨弦扫音色（节奏型） | 电子音色 + 钢弦吉他音色双态切换（含和弦块 Karplus-Strong） |
| 跨设备 | 数据仅存本地浏览器 | 可通过 zip 迁移 |

## 7. 假设与风险

### 7.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|------|---------|-----------|
| 用户有跨设备迁移收藏的需求 | 观察导出/导入使用率 | 降低导出操作门槛，若使用率低则仅做手动备份功能 |

### 7.2 已知风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 录音文件较大导致 zip 包体积过大 | 中 | 低 | 不做体积限制，打包时在 UI 上给出大概预计大小 |
| IndexedDB 空间不足导致导入失败 | 低 | 中 | 导入前检查 IndexedDB 剩余配额，不足时提示用户清理 |
| Karplus-Strong 合成音色在部分设备上性能不足 | 低 | 中 | 降级到简化的噪声+滤波方案 |
