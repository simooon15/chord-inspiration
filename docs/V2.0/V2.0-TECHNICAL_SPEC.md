## 2. 类型系统 (`src/types.ts`)

### 2.1 ChordQuality

和弦品质枚举，13 种。每个品质对应一组固定的半音偏移。

```typescript
export type ChordQuality =
  | 'Major'      // 大三和弦     [0, 4, 7]
  | 'm'          // 小三和弦     [0, 3, 7]
  | '7'          // 属七和弦     [0, 4, 7, 10]
  | 'maj7'       // 大七和弦     [0, 4, 7, 11]
  | 'm7'         // 小七和弦     [0, 3, 7, 10]
  | 'm9'         // 小九和弦     [0, 3, 7, 10, 14]
  | 'maj9'       // 大九和弦     [0, 4, 7, 11, 14]
  | '9'          // 属九和弦     [0, 4, 7, 10, 14]
  | 'dim'        // 减三和弦     [0, 3, 6]
  | 'dim7'       // 减七和弦     [0, 3, 6, 9]
  | 'm7b5'       // 半减七和弦   [0, 3, 6, 10]
  | '7b9'        // 属七降九和弦 [0, 4, 7, 10, 13]
  | '13';        // 属十三和弦   [0, 4, 7, 10, 9] (注意：9 = 21 mod 12 的压缩表示)
```

**关键设计：** `'13'` 的偏移数组实际为 `[0, 4, 7, 10, 9]`，其中 `9` 代表 13 度音（21 半音）在 MIDI 八度内的压缩表示。这在 MIDI 音符计算中通过基底八度 `baseMidiRoot + offset` 自然映射到正确音高（详见第 3.4 节）。

### 2.4 ChordTemplate

和弦模板，是进行（Progression）的基本单元，存储**相对调性根音**的偏移量。

```typescript
export interface ChordTemplate {
  rootOffset: number;      // 距离调性根音的半音偏移量（0 = 调性根音，5 = IV 级）
  quality: ChordQuality;   // 和弦品质
  nashvilleDegree: string; // 纳什维尔数字级数（如 "1", "6-", "4", "b7", "b3"）
  romanDegree: string;     // 罗马数字级数（如 "I", "vi", "IV", "bVII", "bIII"）
  description?: string;    // 这个和弦在当前进行中的角色描述（如 "主和弦", "属和弦"）
}
```

**设计要点：**
- `rootOffset` 是跨调性的抽象偏移量，在运行时通过 `(key.semitones + rootOffset) % 12` 转换为具体物理半音
- `nashvilleDegree` 中的 `-` 后缀表示小调（如 `6-` = vi 级小和弦），`b` 前缀表示降级（如 `b7` = 降七级）
- `description` 用于 UI 中显示每个和弦的情感角色提示

## 3. 乐理转调引擎 (`src/utils.ts`)

### 3.1 半音阶常量

系统使用两套半音阶数组分别对应升号优先和降号优先拼写：

```typescript
export const SHARP_SCALE = ['C', 'C#', 'D', 'D#', 'E', 'F', 'F#', 'G', 'G#', 'A', 'A#', 'B'];
export const FLAT_SCALE  = ['C', 'Db', 'D', 'Eb', 'E', 'F', 'Gb', 'G', 'Ab', 'A', 'Bb', 'B'];
```

物理基准：C = 0。

### 3.2 和弦品质偏移公式

```typescript
export const CHORD_FORMULAS: Record<ChordQuality, number[]> = {
  'Major': [0, 4, 7],
  'm':     [0, 3, 7],
  '7':     [0, 4, 7, 10],
  'maj7':  [0, 4, 7, 11],
  'm7':    [0, 3, 7, 10],
  'm9':    [0, 3, 7, 10, 14],
  'maj9':  [0, 4, 7, 11, 14],
  '9':     [0, 4, 7, 10, 14],
  'dim':   [0, 3, 6],
  'dim7':  [0, 3, 6, 9],
  'm7b5':  [0, 3, 6, 10],
  '7b9':   [0, 4, 7, 10, 13],
  '13':    [0, 4, 7, 10, 9]   // 9 = 21 mod 12（十三度 = 八度 + 六度）
};
```

偏移量相对于和弦根音（即 `rootOffset` 映射后的绝对音名）。

### 3.3 和弦后缀映射

```typescript
export const CHORD_SUFFIXES: Record<ChordQuality, string> = {
  'Major': '',
  'm': 'm',
  '7': '7',
  'maj7': 'maj7',
  'm7': 'm7',
  'm9': 'm9',
  'maj9': 'maj9',
  '9': '9',
  'dim': 'dim',
  'dim7': 'dim7',
  'm7b5': 'm7b5',
  '7b9': '7♭9',      // 使用 Unicode 降号
  '13': '13'
};
```

### 3.4 核心转调算法: `transposeChord()`

```typescript
function transposeChord(template: ChordTemplate, key: KeyConfig): TransposedChord
```

**算法步骤：**

**Step 1 — 计算绝对根音索引：**
```typescript
const rootIndex = (key.semitones + template.rootOffset) % 12;
```

**Step 2 — 确定拼写极性（升号/降号）：**
遵守以下优先级规则：
1. 若级数标记以 `b` 开头（如 `bVII`、`b3`），强制降号拼写
2. 若级数标记包含 `#`，强制升号拼写
3. 否则，以调性的 `preferFlat` 为准

```typescript
const scaleDegreeHasFlat = template.nashvilleDegree.includes('b') || template.romanDegree.startsWith('b');
const scaleDegreeHasSharp = template.nashvilleDegree.includes('#') || template.romanDegree.includes('#');
const prefersFlat = (key.preferFlat || scaleDegreeHasFlat) && !scaleDegreeHasSharp;
const rootName = prefersFlat ? FLAT_SCALE[rootIndex] : SHARP_SCALE[rootIndex];
```

**Step 3 — 拼写和弦名：**
```typescript
const suffix = CHORD_SUFFIXES[template.quality];
const chordName = `${rootName}${suffix}`;
```

**Step 4 — 拼写构成音：**
构成音的拼写尺度选择规则：如果和弦根音本身是降号，构成音也用降号表。
```typescript
const isChordRootFlat = rootName.includes('b') || (key.preferFlat && !rootName.includes('#'));
const noteSpellingScale = isChordRootFlat ? FLAT_SCALE : SHARP_SCALE;

const memberNotes = offsets.map(offset => {
  const noteIndex = (rootIndex + offset) % 12;
  return noteSpellingScale[noteIndex];
});
```

**Step 5 — 生成 MIDI 音符：**
采用中央 C 附近的三八度布局：
```typescript
const baseMidiRoot = 48 + rootIndex;  // C3 = MIDI 48
const midiNotes = offsets.map(offset => baseMidiRoot + offset);
```

**Step 6 — 计算频率：**
```typescript
const rootFrequency = 440 * Math.pow(2, ((baseMidiRoot + (offsets[1] || 4) - 69) / 12));
```

**返回值：** `TransposedChord` 对象（见 2.6 节）。

## 5. 音频合成引擎 (`src/audio.ts`)

### 5.1 架构概览

`AudioSynth` 类，单例导出：

```typescript
class AudioSynth {
  private ctx: AudioContext | null = null;
  private activeNodes: { osc: OscillatorNode; gain: GainNode }[] = [];
  private sequenceIntervalId: number | null = null;
}

export const synth = new AudioSynth();
```

### 5.2 关键参数表

| 参数 | 值 | 说明 |
|------|-----|------|
| 波形 | `triangle` | Rhodes 三角波，温暖偶次谐波 |
| 低通截止频率 | 800 Hz | 过滤刺耳高频 |
| 低通 Q 值 | 1.0 | 轻微共振模拟腔体 |
| 单音最大音量 | `0.15 / max(3, noteCount)` | 均分配防止爆音 |
| Attack | 0.1s (linear ramp) | 极速起音 |
| Release | 0.5s (exponential) | 指数衰减消噪 |
| 扫弦延迟 | 0.06s / 音 | 模拟手指错落感 |
| 单音播放时长 | 1.4s (点击) / 1.8s (循环) | 取决于上下文 |
| MIDI 安全范围 | 36~80 | 低于 36 升八度，高于 80 降八度 |

### 5.3 音频信号链

```
OscillatorNode (triangle) ──┐
OscillatorNode (triangle) ──┼─► GainNode (ADSR) ──► BiquadFilter (lowpass 800Hz) ──► destination
OscillatorNode (triangle) ──┘
```

多个振荡器节点同时发声（和弦内各音），共享同一个滤波器和 AudioContext。

### 5.4 AudioContext 惰性初始化

```typescript
private getContext(): AudioContext {
  if (!this.ctx) {
    this.ctx = new (window.AudioContext || (window as any).webkitAudioContext)();
  }
  if (this.ctx.state === 'suspended') {
    this.ctx.resume();
  }
  return this.ctx;
}
```

仅在首次调用 `playChord()` 时初始化，兼容浏览器自动播放策略。iOS Safari 需要用户手势触发才能启动 AudioContext。

### 5.5 MIDI 频率转换

```typescript
private midiToFreq(note: number): number {
  return 440 * Math.pow(2, (note - 69) / 12);
}
```

A4 (MIDI 69) = 440Hz。

### 5.6 ADSR 包络实现

```typescript
// 步骤 1: 多音均分音量，防止 clipping
const maxVolume = 0.15 / Math.max(3, midiNotes.length);

// 步骤 2: 计算延迟起始时间（扫弦效果）
const noteDelay = strum ? index * 0.06 : 0;
const noteStartTime = now + noteDelay;

gainNode.gain.setValueAtTime(0, now);                                    // 起始静音
gainNode.gain.linearRampToValueAtTime(maxVolume, noteStartTime + 0.1);    // A: Attack 0.1s
gainNode.gain.setValueAtTime(maxVolume, noteStartTime + duration - 0.5);  // D/S: Sustain
gainNode.gain.exponentialRampToValueAtTime(0.0001, noteStartTime + duration); // R: Release 0.5s
```

**防 click 关键：** Release 阶段使用 `exponentialRampToValueAtTime`（指数衰减到接近 0），而非线性衰减或直接停止。指数衰减比线性更平滑，避免波形在非零电平切断产生咔哒声。

### 5.7 单音播放 `playChord()`

```typescript
public playChord(midiNotes: number[], duration = 2.0, strum = true): void
```

1. 获取 AudioContext（惰性初始化）
2. 创建 BiquadFilter (lowpass, 800Hz, Q=1.0)，连接到 destination
3. 对每个 MIDI note:
   a. 创建 OscillatorNode (triangle)
   b. 创建 GainNode
   c. 确保 MIDI 在安全范围 36~80
   d. 设置频率
   e. 配置 ADSR 包络
   f. `osc.connect(gainNode)` → `gainNode.connect(filter)`
   g. `osc.start()` / `osc.stop()` 带延迟
   h. 将节点对加入 `activeNodes` 以便清理
   i. setTimeout 清除引用（duration + 缓冲）

### 5.8 进行循环播放 `playProgression()`

```typescript
public playProgression(
  progressions: number[][],    // MIDI notes 数组的数组
  chordDurSec = 1.8,           // 每和弦播放秒数
  onChordActive: (index: number) => void  // 回调，当前播放和弦索引
): void
```

1. 先 `stopAll()` 清理
2. 用 `setInterval` 以 `chordDurSec * 1000` 间隔依次播放
3. 循环播放：播到末尾后 `currentIndex = 0` 重新开始
4. 每次播放调用 `playChord(progressions[currentIndex], chordDurSec, true)`
5. 通过 `onChordActive` 回调通知 UI 高亮当前和弦

### 5.9 停止 `stopAll()`

```typescript
public stopAll(): void
```

1. `clearInterval(sequenceIntervalId)` 停止循环定时器
2. 遍历 `activeNodes`，对每个节点对调用 `osc.stop()`, `osc.disconnect()`, `gain.disconnect()`
3. 清空 `activeNodes = []`
4. 包含 try-catch 防止对已停止节点重复操作

## 6. SVG 吉他指法渲染系统

### 6.1 吉他指法数据库 (`src/guitarChords.ts`)

**数据结构：**
```typescript
interface GuitarChordsMeta {
  chordName: string;
  frets: (number | 'x')[];  // 6 弦 → 1 弦的按品，'x' = 闷音
  baseFret: number;          // 基准品（显示在弦图左侧）
}
```

**数据规模：** 硬编码指法字典约 140 条，覆盖 12 个根音 × 各品质（Major, m, 7, maj7, m7, 9, maj9, m9, dim, dim7, m7b5, 7b9 等常用手型）。

**指法查找函数 `getGuitarChordFingering(chordName)`:**
1. 规范化和弦名（将 `♭` 转 `b`，`♯` 转 `#`）
2. 在硬编码字典中精确匹配
3. 若未找到，检查同音异名等价键（C# ↔ Db, D# ↔ Eb, F# ↔ Gb, G# ↔ Ab, A# ↔ Bb）
4. 若仍找不到，返回 Em 安全形状 `[0, 2, 2, 0, 0, 0]` 作 fallback

**baseFret 计算算法：**
1. 提取所有数字品值 > 0 的项，取 `minFret` 和 `maxFret`
2. 若 `maxFret > 4` 且 `minFret > 1`：
   - `baseFret = minFret`
   - 在弦图左侧显示 `{baseFret}f` 标签
3. 否则：`baseFret = 1`（显示标准弦枕）

**视口与尺寸：**

- 容器 `98 × 106px`

## 8. 录音系统

### 8.1 录音流程

```
用户点击录音按钮
    │
    ▼
handleStop() ──── 停止当前合成器发声
    │
    ▼
navigator.mediaDevices.getUserMedia({ audio: true })
    │
    ▼
new MediaRecorder(stream)
    │
    ▼
recorder.start() ──── 开始录制
    │
    ▼
每 interval 1000ms: recordingTime++
    │
    ▼
用户点击停止
    │
    ▼
recorder.stop()
    │
    ▼
recorder.onstop:
  1. Blob → saveAudioMemo(id, blob) 写入 IndexedDB
  2. 生成 AudioMemoMeta → prepend recordingList
  3. 更新 localStorage('and_xian_audio_memos')
  4. stream.getTracks().forEach(t → t.stop()) 释放麦克风
```

### 8.2 MediaRecorder 配置

- MIME type: `audio/webm;codecs=opus`（Blob 拼接时未指定，使用默认格式）
- Chunks: `ondataavailable` 收集到 `audioChunksRef`

### 8.3 录音播放

```typescript
const blob = await getAudioMemo(id);
const audioUrl = URL.createObjectURL(blob);
const audio = new Audio(audioUrl);
audio.play();
// cleanup: URL.revokeObjectURL on ended/error
```

### 8.4 麦克风安全释放规范（必须遵守）

录制停止后，必须显式释放硬件麦克风，否则：
1. 浏览器地址栏持续显示麦克风指示灯
2. 耗电并占用音频设备资源

**强制模式：**
```typescript
recorder.onstop = () => {
  // ... 保存 Blob 和元数据 ...

  // 物理停止硬件，释放设备灯
  stream.getTracks().forEach(track => track.stop());
};
```

## 11. 调试、边界条件与安全规范

### 11.1 浏览器兼容

- 所有现代浏览器（Chrome 90+, Firefox 90+, Safari 15+, Edge 90+）
- AudioContext 惰性初始化，首次用户交互后才创建
- Safari 隐私模式 IndexedDB 返回 `SecurityError` 时降级使用内存 fallback
- 使用 `env(safe-area-inset-*)` 适配刘海屏和 Home Indicator

### 11.2 边界处理

| 场景 | 处理方式 |
|------|---------|
| 风格选中后无此风格进行 | 随机池为空时 `return`，不更新 |
| 单条进行仅剩一个 | 跳过"避免重复"逻辑 |
| AudioContext blocked by autoplay policy | 调用 `resume()`，suspended 状态下用户点击触发 |
| IndexedDB 写入失败 | 捕获 error，控制台 log 但不阻塞 UI |
| 录音权限拒绝 | `try-catch` 捕获，alert 提示 |
| 指法未找到 | fallback 到 Em 安全形状 `[0,2,2,0,0,0]`，不崩溃 |
| 播放状态切换（风格/调性变更） | 自动调用 `handleStop()` 停止播放 |
| 录音状态切换进行 | 自动调用 `handleStop()` 停止合成器 |
| MIDI 范围越界 | 低于 36 升八度，高于 80 降八度 |
| 13 和弦偏移量 9 | 在 MIDI 计算中通过基底八度自然扩展，无需特殊处理 |

### 11.3 安全规范

1. **AudioContext 惰性初始化**：不在构造函数中创建，首次 `getContext()` 时才实例化
2. **麦克风释放**：`recorder.onstop` 中强制调用 `stream.getTracks().forEach(t => t.stop())`
3. **节点清理**：`stopAll()` 遍历清除所有活跃节点，`try-catch` 包裹防止重复停止异常
4. **CSS 安全区**：使用 `env(safe-area-inset-*)` 避免刘海屏遮挡
