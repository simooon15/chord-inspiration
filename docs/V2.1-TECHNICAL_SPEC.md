## 2. 节奏型数据层

### 2.1 类型定义 (`src/rhythm/types.ts`)

```typescript
export type TimeSignature = '4/4' | '3/4' | '6/8';
export type StrumDirection = 'down' | 'up';
export type StrumSlot = StrumDirection | null;

export interface BeatGrid {
  subdivision: number;        // 每拍子分格数，4 = 十六分音符
  slots: StrumSlot[];         // 长度 === subdivision，每个格子的扫弦方向
}

export interface RhythmPattern {
  id: string;                 // 唯一标识，如 "folk_basic"
  name: string;               // 中文名，如 "民谣基础扫弦"
  timeSignature: TimeSignature;
  beats: BeatGrid[];          // 长度 = beatsPerBar × barCount
  barCount: number;           // 1 = 单小节, 2 = 两小节
  // beats 数组长度示例：
  //   4/4, barCount=1 → beats[4]
  //   4/4, barCount=2 → beats[8]
  //   3/4, barCount=1 → beats[3]
  //   6/8, barCount=1 → beats[6]
}
```

## 3. 播放调度引擎 (`src/rhythm/engine.ts`)

### 3.1 职责

替换 V2.0 的 `setInterval(chordDuration)` 简单循环，升级为以 BPM 为时钟源、按格子 tick 推进的调度器。

### 3.2 核心接口

```typescript
export class RhythmEngine {
  private bpm: number;
  private timeSignature: TimeSignature;
  private pattern: RhythmPattern | null;
  private metronomeOn: boolean;
  private rhythmOn: boolean;
  private chordsOn: boolean;
  private chordMode: 'pattern' | 'bar';  // 和弦切换模式
  private tickIntervalId: number | null;
  private currentBeat: number;     // 当前拍号索引 (0-based)
  private currentSlot: number;     // 当前格子索引 (0-based)
  private currentChordIndex: number;
  private tickCount: number;       // 总 tick 数（用于预备拍计数）

  // 回调
  onTick?: (info: TickInfo) => void;       // 每个 tick 触发
  onChordChange?: (index: number) => void; // 和弦切换时触发
  onCountInEnd?: () => void;               // 预备拍结束
}
```

### 3.3 Tick 时钟

```
tickInterval = (60 / BPM / subdivision) × 1000 ms
```

- 4/4, subdivision=4: interval = 60000 / BPM / 4 ms
- BPM=100 时，interval = 150ms（每个十六分音符 150ms）
- BPM=200 时，interval = 75ms
- BPM=40 时，interval = 375ms

**时钟精度校准：** 使用 `performance.now()` 记录起始时间戳，每 tick 计算实际经过时间与理论时间的偏差，累计超过半个 tick 时补跳一个 tick。

```typescript
private scheduleNextTick(): void {
  const expectedTickTime = this.tickCount * this.tickIntervalMs;
  const actualElapsed = performance.now() - this.startTime;
  const drift = actualElapsed - expectedTickTime;

  // 漂移补偿：跳过或延迟
  const adjustedDelay = Math.max(0, this.tickIntervalMs - drift);
  this.tickTimerId = window.setTimeout(() => this.fireTick(), adjustedDelay);
}
```

### 3.4 Tick 事件处理

每个 tick 依次执行，三个音频层独立触发互不干扰：

```
fireTick()
  ├─ 预备拍阶段 (tickCount < ticksPerBar)：
  │    └─ 节拍器 ON → 播放 click（强拍/弱拍）
  │
  └─ 正常播放阶段 (tickCount >= ticksPerBar)：
       ├─ slot === 0 → 节拍器 ON → 播放 click（强拍/弱拍）
       ├─ 节奏型 ON && slots[slot] !== null → 播放扫弦
       │    ├─ 和弦 ON → StrumAudio.playStrum(midiNotes, direction)
       │    └─ 和弦 OFF → StrumAudio.playMutedStrum(direction)
       └─ 和弦切换判断：
            ├─ chordMode === 'pattern' → 每 beatInBar × barCount 拍换和弦
            └─ chordMode === 'bar' → 每 beatInBar 拍换和弦
       └─ currentSlot++, 回卷 currentBeat
```

### 3.5 启动/停止

```typescript
start(): void {
  this.stop();
  this.startTime = performance.now();
  this.tickCount = 0;
  this.currentBeat = 0;
  this.currentSlot = 0;
  this.currentChordIndex = 0;
  this.scheduleNextTick();
}

stop(): void {
  if (this.tickTimerId) clearTimeout(this.tickTimerId);
  // 停止所有活跃的扫弦节点
  this.strumAudio.stopAll();
}
```

## 4. 音频层改动 (`src/audio.ts`)

### 4.0 spessasynth_lib 初始化

使用 spessasynth_lib 的 `Synthetizer` 加载精简版 SoundFont 文件：

```typescript
import { Synthetizer } from 'spessasynth_lib';

let synth: Synthetizer | null = null;

async function initGuitarSampler(): Promise<void> {
  const ctx = getContext();                      // 复用 V2.0 的 AudioContext
  await ctx.audioWorklet.addModule(
    'https://cdn.jsdelivr.net/npm/spessasynth_lib@3.26.32/dist/worklet_processor.min.js'
  );
  const sf2Data = await fetch('/samples/Guitar_Acoustic_Stripped.sf2').then(r => r.arrayBuffer());
  synth = new Synthetizer(ctx.destination, sf2Data);
  await synth.waitForReady();
}
```

**初始化时机**：在用户第一次与节奏面板交互时触发（点击播放/点击开关），不阻塞应用启动。加载期间 playClick 仍然可用。

### 4.1 节拍器音色 (`playClick`)

使用白噪声脉冲包络模拟木鱼声（代替此前设计的正弦波 click），不经过 spessasynth_lib：

```typescript
playClick(isStrongBeat: boolean, volumeMultiplier = 1.0): void {
  const ctx = this.getContext();
  const now = ctx.currentTime;

  // 木鱼声：白噪声短脉冲 + 带通滤波器塑造音色
  const bufferSize = Math.floor(ctx.sampleRate * 0.05); // 50ms
  const buffer = ctx.createBuffer(1, bufferSize, ctx.sampleRate);
  const data = buffer.getChannelData(0);
  for (let i = 0; i < bufferSize; i++) data[i] = Math.random() * 2 - 1;

  const source = ctx.createBufferSource();
  source.buffer = buffer;

  const filter = ctx.createBiquadFilter();
  filter.type = 'bandpass';
  filter.frequency.value = isStrongBeat ? 1800 : 1200;     // 强拍更亮
  filter.Q.value = 0.5;

  const vol = (isStrongBeat ? 0.25 : 0.15) * volumeMultiplier;
  const gain = ctx.createGain();
  gain.gain.setValueAtTime(vol, now);
  gain.gain.exponentialRampToValueAtTime(0.001, now + 0.04); // Decay 40ms

  source.connect(filter).connect(gain).connect(ctx.destination);
  source.start(now);
  source.stop(now + 0.06);
}
```

**模式自适应音量：**
- 纯节拍器模式（和弦 OFF）：`volumeMultiplier = 1.3`
- 混合模式（和弦 ON）：`volumeMultiplier = 0.8`

### 4.2 扫弦音色 (`playStrum`) — spessasynth_lib 真人采样

扫弦使用 spessasynth_lib 的 `Synthetizer.noteOn()` 触发 SoundFont 内的吉他采样，取代三角波合成：

```typescript
async ensureGuitarSynth(): Promise<void> {
  if (synth) return;
  await initGuitarSampler();
}

playStrum(midiNotes: number[], direction: StrumDirection): void {
  if (!synth) return;

  if (direction === 'down') {
    // 6 弦从低→高，12ms/弦，声相左→右
    midiNotes.sort((a, b) => a - b).forEach((note, i) => {
      setTimeout(() => synth.noteOn(0, note, 80), i * 12);
    });
  } else {
    // 仅高音 3 弦，8ms/弦，音量 70%
    const highNotes = [...midiNotes].sort((a, b) => b - a).slice(0, 3);
    highNotes.forEach((note, i) => {
      setTimeout(() => synth.noteOn(0, note, 56), i * 8);  // 80×0.7≈56
    });
  }
}
```

### 4.3 闷音扫弦 (`playMutedStrum`)

使用 SF2 的 "Open Muted" 预设：

```typescript
playMutedStrum(): void {
  if (!synth) return;
  // 使用 SF2 中 Open Muted 预设的 MIDI 音高（Bank=0, Program=35）
  const mutedNotes = [48, 52, 55, 60, 64, 67];
  mutedNotes.forEach((note, i) => {
    setTimeout(() => synth.noteOn(0, note, 40), i * 8);
  });
}
```

### 4.4 和弦播放 (`playChord`) — V2.1 也使用 SF2

V2.1 中 `playChord()` 在节奏型模式下也使用 spessasynth_lib + SF2 钢弦预设：

```typescript
playChord(midiNotes: number[]): void {
  if (synth) {
    // 使用 SF2 "Guitar Expressive" 预设 (Bank=0, Program=0)
    midiNotes.forEach(note => synth.noteOn(0, note, 70));
  }
}
```

### 4.5 三个音频层的独立性

| 层 | 音源 | 依赖 | 说明 |
|----|------|------|------|
| 节拍器 | 白噪声脉冲（纯 Web Audio） | 无 | 不依赖 SF2，即时可用 |
| 扫弦 | spessasynth_lib + SF2 | SF2 加载完成 | 按 pattern 格子触发 |
| 和弦 | spessasynth_lib + SF2 | SF2 加载完成 | 按格切换 |

三者独立触发：每个 tick 单独判断各开关状态，互不影响。

### 5.2 播放模式路由

播放按钮 `handlePlay()` 的逻辑：

```typescript
function handlePlay() {
  if (isPlaying) {
    engine.stop();
    synth.stopAll();
    setIsPlaying(false);
    return;
  }

  if (rhythmOn) {
    // 使用新的 RhythmEngine
    engine.configure({ bpm, timeSignature, pattern, metronomeOn, rhythmOn, chordsOn });
    engine.onTick = (info) => {
      setCurrentTickBeat(info.beat);
      setCurrentTickSlot(info.slot);
      setActiveChordIndex(info.chordIndex);
    };
    engine.start();
  } else {
    // 使用 V2.0 的 synth.playProgression()
    synth.playProgression(midiArrays, chordDuration, setActiveChordIndex);
  }
  setIsPlaying(true);
}
```

### 5.3 播放中切换参数

如果正在播放中修改 BPM/拍号/pattern，不中断播放，在**下一小节边界**生效：

```typescript
useEffect(() => {
  if (isPlaying && rhythmOn) {
    engine.updateConfig({ bpm, timeSignature, pattern });
  }
}, [bpm, timeSignature, selectedPatternId]);
```

`engine.updateConfig()` 内部标记参数变更，在下一个 `currentBeat === 0 && currentSlot === 0` 时应用。

### 5.4 停止时清理

```typescript
function handleStop() {
  engine.stop();
  synth.stopAll();
  setIsPlaying(false);
  setActiveChordIndex(-1);
  setCurrentTickBeat(0);
  setCurrentTickSlot(0);
}
```

## 7. 预览网格渲染

### 7.1 格子计算

```typescript
// 从选中 pattern 计算总格子数
const ticksPerBar = pattern.beats.reduce((sum, b) => sum + b.subdivision, 0);
// 4/4, sub=4:  ticksPerBar = 16
// 3/4, sub=4:  ticksPerBar = 12
// 6/8, sub=4:  ticksPerBar = 24
```

### 7.2 渲染结构

不创建独立组件文件，在 App.tsx 的播放栏区域直接渲染（与 V2.0 组件组织方式一致）。

使用 `motion.span` 渲染每个格子，当前高亮格子使用 `layoutId="rhythm_cursor"` 实现平滑过渡（可选，也可直接用 `animate` 切换 backgroundColor）。

### 7.3 格子高亮过渡

高亮格子在每 tick 切换。直接使用 `motion.span` 的 `animate` prop 切换 `backgroundColor`，spring 过渡 300ms，视觉上平滑。

## 9. 和弦发声验证

V2.1 在开发前需要验证 V2.0 的和弦音频基座。验证方法：基于现有 `transposeChord()` 和 `playChord()`，编写一次性验证脚本或手动测试清单，遍历 13 种品质 × 12 调，确认 MIDI 音符与理论音高一致。

验证重点：
- `CHORD_FORMULAS` 半音偏移正确性
- `transposeChord()` 移调后拼写和 MIDI notes 正确
- `clampMidi()` 折叠边界（36-80）不产生明显倒置

此验证为一次性开发前置步骤，不产生持久化代码。
