# V2.1.1 PRD — 节奏体验增强

> Z 修复版本：在 V2.1 基础上优化节奏模块 UI/UX 并增强扫弦真实感

## 背景

V2.1 的节奏型模块存在以下体验问题：

1. **UI 布局割裂**：节奏面板与"再换一组"按钮分别独立放置，间距不当，视觉上不属于同一模块组
2. **展开无自动滚动**：点击"节奏"按钮后页面不自动下滑到节奏面板，需手动滚动
3. **展开动画生硬**：节奏面板展开的弹簧阻尼过高（damping=15），缺乏和弦块展开的 Q 弹感
4. **扫弦 MIDI 感**：所有扫弦严格量化在节拍格子上，力度一致，缺乏真人演奏的呼吸感和律动
5. **预览缺乏力度层次**：Pattern 预览网格中所有扫弦显示相同视觉权重，无法反映力度差别
6. **页面标题冗余**：网页标题（`<title>`）包含多余描述

## 目标

在不改变 V2.1 架构的前提下，统一节奏模块的视觉归属、优化交互动效，并为 BeatGrid 新增 3 个可选维度增强扫弦真实感。

## 一、UI/UX 改进

### 1.1 节奏面板独立 + "再换一组"按钮分离

- **现状**：第一次合并时错误地将节奏面板和"再换一组"包入统一卡片，导致两者视觉冲突
- **修正**：节奏面板保持独立卡片（`bg-[#fafdf6]/40 backdrop-blur-md rounded-[22px] border`），"再换一组"保持独立按钮置于其下方
- **间距**：节奏面板展开时，按钮通过 `mt-5` 与面板保持视觉距离；面板收起时按钮无多余上间距

### 1.2 试听控制模块自动回缩

- 打开节奏面板时，试听控制模块的速度滑块自动以弹簧动画回缩
- 关闭节奏面板时，速度滑块自动展开恢复
- 弹簧参数：`stiffness: 105, damping: 11, mass: 0.85`（与和弦块展开一致）
- 速度滑块外的分割线 (`border-t`) 在回缩时自动隐藏，展开时恢复
- 同时将模块间距从 `mb-6` 改为 `mb-4`，移除节奏面板内部的 `mt-3`

### 1.3 展开自动下滑

- 为节奏区域绑定 `useRef`（`rhythmSectionRef`）
- `showRhythmPanel` 变为 `true` 时调用 `scrollIntoView({ behavior: 'smooth', block: 'start' })`
- 节奏面板首次展开也有进入动画（移除了 `AnimatePresence initial={false}`）

### 1.4 展开动效 Q 弹化

- 弹簧参数从 `stiffness: 110, damping: 15` 改为 `stiffness: 105, damping: 11`（与 ChordCard 展开参数一致）
- 更低阻尼 → 回弹更明显，节奏模块展开不再生硬

### 1.5 页面标题简化

- `<title>` 从 "和弦灵感 - 极客 Songwriter 工作台" 改为 "和弦灵感"
- 应用内 `<h1>` 已经为"和弦灵感"，无需改动

## 二、扫弦真实感增强

### 2.1 新增数据维度

#### 2.1.1 力度随机波动 (`variance`)

- **类型**: `number[]`，长度 = subdivision，默认 0
- **含义**: 该 slot 触发时 velocity 在 `[baseVelocity - N, baseVelocity + N]` 范围内随机
- **听感**: 打破完全一致的复读感，每次循环的力度有微小自然变化

#### 2.1.2 弦域控制 (`stringRange`)

- **类型**: `[number, number][]`，长度 = subdivision，默认 `[0, 5]`
- **含义**: 控制该 slot 扫哪几根弦。`[0]` = 低音弦索引，`[1]` = 高音弦索引。`[0,5]`=全扫 6 根，`[0,2]`=只扫低音 3 根，`[3,5]`=只扫高音 3 根
- **听感**: 同一和弦在不同拍子上有厚/薄层次对比，主歌收着弹低音区，副歌全扫炸开

#### 2.1.3 扫弦速度 (`strumSpeed`)

- **类型**: `number[]`，长度 = subdivision，默认 `1.0`
- **含义**: 控制该 slot 扫弦的快慢。`>1.0`=更慢（更松弛），`<1.0`=更快（更急促）
- **听感**: 强调时快速果断，放松时缓慢铺展，直接改变情绪

### 2.2 Pattern 预览力度可视化

根据已有的 `durations` 值渲染不同视觉权重：

| durations 值 | 显示 | 说明 |
|-------------|------|------|
| 1 (基准) | `↓` / `↑` 正常字重 | 普通扫弦 |
| 2 (中重音) | `↓` / `↑` 加粗 (`font-bold`) | 重拍强调 |
| 3 (强重音) | `⇓` / `⇑` 加粗 | 强重音 |
| 4 (超强) | `⇓` / `⇑` 加粗 + 深色 | 超强重音，起手/结尾 |

- 空 slot 保持 `·` 不变
- 播放中高亮当前 slot 时叠加播放状态色

## 三、技术方案

### 3.1 数据模型

```typescript
// src/rhythm/types.ts — BeatGrid 新增字段
interface BeatGrid {
  subdivision: number;
  slots: StrumSlot[];
  durations?: number[];

  // V2.1.1 新增
  variance?: number[];             // ±N，默认 0
  stringRange?: [number, number][]; // 默认 [0,5]
  strumSpeed?: number[];           // 倍率，默认 1.0
}

// 新增：扫弦参数打包对象
interface StrumOptions {
  direction: StrumDirection;
  holdDurationMs: number;
  midiNotes: number[];
  variance: number;
  stringRange: [number, number];
  strumSpeed: number;
}
```

### 3.2 Engine 变更

- `fireTick()`: 从 `beatGrid` 读取新字段，缺失时使用默认值，组装 `StrumOptions`
- `playStrumFromEngine` 签名改为 `playStrumFromEngine(opts: StrumOptions)`

### 3.3 Audio 变更

- `playStrumSF2`: 接收 `StrumOptions`
  - velocity += `randomInt(-variance, +variance)`
  - 弦域控制通过标准吉他弦音高（40, 45, 50, 55, 59, 64）过滤 `midiNotes`
  - 扫弦速度倍率 → `perStringDelay *= strumSpeed`

### 3.4 UI 组件变更

- 新增 `rhythmSectionRef`，展开时 `scrollIntoView`
- 节奏面板保持独立卡片；"再换一组"按钮独立置于下方，展开时通过 `mt-5` 保持间距
- 速度滑块包裹在 `AnimatePresence` 中，开/关节奏面板时以弹簧动画（`stiffness: 105, damping: 11`）自动回缩/展开
- 节奏面板 `AnimatePresence` 移除 `initial={false}`，保证每次展开都有进入动效
- 展开动画弹簧参数改为 `stiffness: 105, damping: 11`
- `<title>` 简化为"和弦灵感"

### 3.5 向后兼容

- 现有 pattern 不填新字段 → 全部使用默认值 → 行为与 V2.1 完全一致
- `slots()` 辅助函数不变

## 四、Pattern 配置策略

V2.1.1 保留两个核心 pattern，手工配置新字段作为示范：

| Pattern | 重点维度 | 说明 |
|---------|---------|------|
| D DDU UD DDU | variance + strumSpeed | 流行扫弦基础律动，力度波动 + 速度变化 |
| The Cure · 经典摇滚律动 | stringRange + variance | 摇滚律动，低音/高音区层次对比 |
