## 1. 产品概述

### 1.1 一句话定位

给「略懂乐理但卡在套路的爱好者」一个不仅能看、还能听和记的完整创作工作台，让灵感从发现到记录一步到位。

### 1.2 目标用户

专业的、完全不懂的，都不需要。核心用户是**中间层**：

- 学过一点乐理，知道和弦走向的概念，听过 1-6-4-5、卡农进行
- 对更丰富的走向没有积累，想创作但不知道从哪下手
- 拿起琴脑子一片空白，弹出来的东西自己都觉得雷同
- 知道「我缺素材」但不知道去哪找
- 弹到喜欢的和弦组合时，有记下来和录下来的冲动

### 1.3 核心价值

**核心闭环：** 打开网页 → 选调性选风格 → 点一下 → 看到一组随机和弦进行 → 试听音色 → 记录灵感 → 收藏管理。

每生成一次，不仅是看一个素材，还能马上开始创作。

**按版本演进：**

| 版本 | 新增核心价值 |
|------|-------------|
| V1.0 | 零摩擦看到随机和弦进行，级数标注让人看懂「为什么这么走」 |
| V2.0 | 可听、可写笔记、可录音、可收藏管理，形成创作闭环 |
| V2.1 | 可调节拍器和扫弦节奏型，让试听从匀速升级为有律动伴奏 |
| V2.2 | 数据可导出备份（zip）；钢弦吉他音色可选，让试听更真实 |

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 生成成功率 | ≥90% | 生成按钮点击 / 页面打开 |
| 重复使用率 | ≥50% | 同一会话内连续生成 ≥2 次的用户占比 |
| 试听使用率 | ≥40% | 生成后点击播放按钮的用户占比 |
| 笔记使用率 | ≥15% | 生成后打开笔记面板的用户占比 |
| 节奏型使用率 | ≥30% | 试听时开启节奏型的用户占比 |
| 节拍器独立使用率 | ≥15% | 仅开启节拍器的用户占比 |
| 操作步数 | 核心流程 ≤3 步 | 逐条验收 |

### 1.5 不做什么

- 不登录、不注册
- 不教乐理课程
- 不接入后端服务，无云同步
- 不集成 AI 生成
- 不做采样类真实乐器音色（除钢弦吉他合成音色外）
- 不支持桌面端独立布局（保持移动端优先）

## 2. 版本演进路线

### 2.1 重大节点

| 版本 | 状态 | 一句话 |
|------|------|--------|
| **V1.0** | 已发布 | 纯静态页，随机和弦生成 + 12 调转调 |
| **V1.0.1** | 已发布 | 响应式适配 375-440px + 间距压缩 + 字体优化 + 点击修复 |
| **V2.0** | 已发布 | 高集成度创作工作台：试听、笔记、录音、收藏、吉他指法 |
| **V2.1** | 开发中 | 节奏型系统：节拍器、扫弦 pattern、三模式协同、方向感知扫弦 |
| **V2.2** | 规划中 | 导出/导入（jsZip）+ 钢弦吉他音色（Karplus-Strong 合成） |
| **V3.0** | 远期 | PWA：离线缓存 + 安装到桌面，类 App 体验 |
| **V4.0** | 远期 | 后端：账号体系 + 云端同步 |
| **V5.0** | 远期 | 独立 App，复用 V4.0 后端 |

### 2.2 演进逻辑

```
Web 功能迭代 → PWA 安装体验 → 数据同步后端 → 跨平台 App
  (V2.1/V2.2)       (V3.0)          (V4.0)         (V5.0)
```

核心原则：重大节点升得慢，大部分功能演进在同一个 X 版本内完成。只有产品形态发生质变时才跳下一个重大节点。

## 3. V1.0

### 3.1 功能范围

| 维度 | V1.0 做 | V1.0 不做 |
|------|---------|-----------|
| 调性 | 12 调选择器，默认 C 调 | 自动检测调性 |
| 风格 | 5 种：流行 / 民谣 / 摇滚 / 爵士 / R&B | 自定义风格、混合风格 |
| 生成 | 选调 + 风格 → 随机一组和弦进行 | 用户输入条件、AI 生成 |
| 展示 | 和弦名 + 构成音 + 级数（纳什维尔数字） | 五线谱、指位图、音频播放 |
| 交互 | 换调实时映射、一键生成、一键刷新 | 撤销、历史记录 |
| 设备 | 移动端竖屏（375-414px） | iPad / 桌面端 |
| 保存 | 无 | 收藏、导出 |

### 3.2 功能详情

#### 3.2.1 调性与风格选择

- 页面顶部调性选择器，12 调可选，默认 C 调
- 调性下方风格选择器，5 种风格可选，默认流行
- 切换调性后已展示的和弦即时映射到新调，不重新生成
- 切换风格后下次生成时生效

#### 3.2.2 和弦生成（「给我灵感」）

- 点击后根据当前调性和风格，随机抽取一组和弦进行
- 防连续重复：确保与上一次不完全相同

#### 3.2.3 和弦卡片展示

- 每张卡片三列布局：和弦名（左）｜ 构成音（中）｜ 级数（右）
- 四张卡片阶梯式入场动画
- 点击卡片有缩放反馈

#### 3.2.4 「再换一组」按钮

- 保持当前调性和风格，重新随机生成一组

### 3.3 优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 随机生成 + 调性 + 级数展示 | 没有生成就没产品 |
| P1 | 5 种风格 × 各 ≥15 组数据 | 数据不够用户用一次就腻 |
| P2 | 核心操作 ≤3 步 | 所有功能的约束条件 |

## 4. V1.0.1

### 4.1 变更背景

V1.0 以 `max-width: 414px` 固定宽度开发和测试，上线后出现四个问题：

| # | 问题 |
|---|------|
| 1 | 414px 固定容器在 iPhone 17（402px）和 17 Pro Max（440px）下不适配 |
| 2 | 固定间距和 padding 导致总高度过大，需要滚动才能看完全部信息 |
| 3 | 标题使用系统 sans-serif，与和弦卡片的衬线字体缺乏层次对比 |
| 4 | 和弦卡片入场动画与点击缩放冲突，导致连续点击卡顿 |

### 4.2 变更范围

| 维度 | V1.0.1 做 |
|------|-----------|
| 布局 | 容器宽度自适应 375-440px，内边距用 viewport 相对单位 |
| 间距 | 缩小顶部留白和区块间距，减少总高度 |
| 字体 | 标题改用思源宋体 |
| 动效 | 修复和弦卡片点击卡顿 |

覆盖机型：iPhone SE（375px）到 iPhone 17 Pro Max（440px）。

### 4.3 功能详情

#### 4.3.1 容器自适应

取消固定宽度，改为 `max-width: 440px` 弹性容器，横向内边距随屏幕自适应。

#### 4.3.2 纵向间距压缩

总计减少约 50-60px 纵向空间，iPhone SE 竖屏可一屏看完核心信息。

#### 4.3.3 标题字体

标题「和弦灵感」从系统无衬线体改为思源宋体（Noto Serif SC），提升品牌层次感。

#### 4.3.4 和弦卡片点击优化

入场动画完成后释放 transform 属性，解决 WebKit 合成器中 CSS animation 与 `:active` 伪类的优先级冲突。

## 5. V2.0

### 5.1 变更背景

V1.0 只是「能看」的随机和弦生成器——纯静态页，零依赖。上线后暴露三个硬伤：只能看不能听（用户必须拿琴试才知道好不好听）、灵感无法留存（歌词联想转瞬即逝）、技术架构到天花板（加音频合成/录音存储/动画交互需要工程化体系）。

V1.0.1 做了一次响应式适配的修补，确认了静态页路线的边界。V2.0 选择 Vite + React 19 + TypeScript 全量重构，不是为了追新技术，而是为了解锁四个 V1.0 做不到的能力：试听、指法、笔记、录音。

### 5.2 功能范围

| 维度 | V2.0 做 | V2.0 不做 |
|------|---------|-----------|
| 调性 | 12 调选择器，默认 C 调，切换即实时映射 | 自动检测调性 |
| 风格 | 5 种：流行 / 民谣 / 摇滚 / 爵士 / R&B，各 ≥15 组 | 自定义风格、混合风格 |
| 生成 | 选调 + 风格 → 随机一组和弦进行，防连续重复 | AI 生成、用户输入条件 |
| 展示 | 和弦名 + 构成音 + 级数（纳什维尔/罗马双模式） | 五线谱、MIDI 导出 |
| 试听 | Web Audio 合成器，循环播放，速度可调 | 采样音色、多乐器选择 |
| 指法 | 吉他 6 弦 5 品弦图，横按自适应偏移 | 钢琴指法、尤克里里指法 |
| 笔记 | 每个进行绑随笔文本框，自动存盘 | 富文本编辑、云端同步 |
| 录音 | MediaRecorder → 本地存储，播放/下载/删除 | 录音编辑、波形可视化 |
| 收藏 | 收藏夹侧滑抽屉，行内笔记编辑器 + 录音管理器，可载入使用 | 排序、搜索、分类 |
| 乐理帮助 | 内嵌说明弹窗（级数概念、双记谱法、转调原理） | 交互式乐理教程 |
| 设备 | 移动端竖屏（375-440px） | iPad / 桌面端 |
| 保存 | LocalStorage + IndexedDB 本地持久化 | 跨设备同步 |

### 5.3 功能详情

#### 5.3.1 调性与风格选择器

- 顶部并排双选择器，毛玻璃圆角胶囊样式
- 调性：12 调（C / Db / D / Eb / E / F / F# / G / Ab / A / Bb / B），默认 C
- 风格：5 种（流行 / 民谣 / 摇滚 / 爵士 / R&B），各 ≥15 组进行数据，含爵士特征和弦（七/九和弦）
- 切换调性后已有和弦即时映射到新调，不重新生成
- 切换风格后立即停止当前播放

#### 5.3.2 和弦生成

- 点击后根据当前调性和风格，随机抽取一组和弦进行
- 默认首次打开时预渲染一组，无空状态
- 防连续重复：确保与上一次不完全相同

#### 5.3.3 和弦卡片区

- 阶梯式动画由左至右滑入
- 每张卡片三列布局：和弦名（左）｜ 构成音（中）｜ 级数（右）
- 点击卡片 = 触发单音试听 + 展开/收起吉他指法图
- 播放过程中的当前和弦高亮

#### 5.3.4 记谱模式切换

- 播放控制栏行内切换按钮
- 纳什维尔数字（1-5-6-4）↔ 罗马记号（I-V-vi-IV）
- 切换即时生效，不重新生成

#### 5.3.5 试听系统

- Web Audio 合成器，三角形波 + 低通滤波，模拟 Rhodes 电钢琴暖感
- 循环播放：按设定速度依次弹出每个和弦的音，当前播放和弦卡片高亮
- 速度可调（每级和弦持续时间）
- 单和弦试听：点击和弦卡片弹出该和弦的琶音
- 播放/停止按钮（播放时红色「停止试听」/ 停止时绿色「循环播放」）

#### 5.3.6 吉他指法图

- 点击和弦卡片展开/收起
- 6 弦 5 品 SVG 弦图，显示按指位置
- 横按自适应：高品位和弦自动偏移并标注基准品位
- 右侧显示 TAB 文本和构成音名
- 覆盖所有 12 调 × 常见品质

#### 5.3.7 创作面板

- 点击进行信息卡展开双标签面板
- **随手笔记**：textarea 绑定当前进行，输入即自动存储，显示「已自动存盘」
- **哼唱小样**：
  - 圆形麦克风按钮，点击开始/停止录音
  - 录音时显示实时计时
  - 录制完成后保存到本地，可播放、下载、删除
  - 录音停止后自动释放麦克风

#### 5.3.8 收藏夹工作台

- 入口：顶部 header 右侧文件夹心形图标
- 打开方式：从底部弹出的侧滑抽屉
- 每项显示：级数公式 + 风格标签，当前已载入的标注「已载入」
- 行内操作：随笔手记、哼唱小样管理、载入使用、取消收藏
- 空状态：无收藏时显示提示文案

#### 5.3.9 「再换一组」按钮

- 主面板底部，保持当前调性和风格，重新随机生成一组

#### 5.3.10 乐理说明弹窗

- 顶部 header 右侧问号图标触发
- 居中弹出模态框
- 内容：什么是和弦级数、纳什维尔数字与罗马记号的区别、转调原理

### 5.4 优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 随机生成 + 调性 + 级数展示 + 记谱模式切换 | 没有核心生成就没有产品 |
| P1 | 5 种风格 × 各 ≥15 组数据 | 数据不够用户用一次就腻 |
| P1 | Web Audio 试听（循环播放 + 速度控制） | 区分于 V1.0 的核心卖点 |
| P2 | 吉他指法图 | 为弹奏提供直接参考 |
| P2 | 创作面板（随手笔记 + 哼唱录音） | 心流延续的关键闭环 |
| P3 | 收藏夹工作台（含行内编辑） | 增强回访粘性 |
| P3 | 乐理说明弹窗 | 降低入门门槛 |

## 6. V2.1 节奏型系统

> 详细内容见 [V2.1-PRD](./V2.1-PRD.md)

### 6.1 变更背景

V2.0 完成了创作工作台闭环，但试听体验有两个明显不足：试听是「平的」——匀速琶音展示不了律动，用户听到一组进行后脑子里浮现的是有律动的伴奏，但 V2.0 只给机械的琶音；练琴场景缺位——用户拿着琴想跟着弹，但没有节拍器，只能另开 App。

V2.1 的节奏型系统改变了试听的产品形态——从匀速琶音升级为有律动的完整伴奏。节拍器底座让它在练琴场景下独立可用。在 V2.0 的工程基座上增量构建，不改变技术架构。

### 6.2 核心功能

- **节拍器底座（P0）**：BPM 40-200 可调，4/4、3/4、6/8 拍号，强拍区分，独立开关
- **节奏型数据模型（P0）**：15 套内置扫弦 pattern（4/4 11 套 + 3/4 2 套 + 6/8 2 套）
- **三开关模式（P0）**：节拍器/节奏型/和弦独立组合，四种典型场景
- **方向感知扫弦（P1）**：↓ 低→高、↑ 高→低，音频区分
- **节奏面板 UI（P1）**：折叠面板，三开关行 → BPM 滑块 → 拍号选择 → 下拉选 pattern → 预览网格
- **预备拍（P2）**：1 小节 count-in
- **播放高亮（P3）**：网格当前格子高亮

### 6.3 不做什么

- 不做自定义节奏型编辑器、多小节节奏型、按风格分类、按拍切换和弦

## 7. V2.2 导出/导入 + 音色

> 详细内容见 [V2.2-PRD](./V2.2-PRD.md)

### 7.1 变更背景

V2.1 解决了节奏型问题，但两个痛点仍在：数据困在浏览器里——V2.0/V2.1 的收藏、笔记、录音全存在 LocalStorage/IndexedDB 里，清缓存就没了，换设备也带不走；音色单一——三角波合成器虽好听，但用户用这个工具来辅助吉他创作，自然希望听到更接近吉他的音色。

V2.2 并行推进两个模块：导出/导入解决数据安全感，钢弦吉他音色提升聆听体验。两者在 V2.0/V2.1 工程基座上增量构建。

### 7.2 核心功能

- **导出收藏（P1）**：管理模式多选，jsZip 打包 .zip，每个进行独立文件夹（chords.json + chords.md + recordings/）
- **导入收藏（P1）**：选 .zip → 预览弹窗 → 冲突处理（覆盖/跳过/保留两份）→ 写入存储
- **钢弦吉他音色（P2）**：Karplus-Strong 合成，播放栏电子 ↔ 吉他双态切换

### 7.3 不做什么

- 不做云端同步（V4.0）、增量同步、录音格式转换、更多乐器音色

## 9. 全局验收标准

### US-1 (P0)：随机生成和弦进行

**As a** 会弹点吉他/键盘但不知道弹什么的爱好者
**I want to** 选个调性和风格，获得一组随机的和弦进行
**so that** 能立刻开始弹，不打断创作心流

- **首次使用**：打开网页，调性默认 C、风格默认流行，页面已预渲染一组和弦进行
- **连续生成不重复**：每次生成与上一次不完全相同
- **调性实时映射**：切换调性后和弦名和构成音即时变化

### US-2 (P1)：风格差异化

**As a** 写了几首歌都觉得一个味的唱作人
**I want to** 在不同风格间切换并得到不同的和弦类型
**so that** 接触不熟悉的进行，写出更多样的作品

- **风格差异**：爵士包含七/九和弦，与流行明显不同
- **同风格多样性**：同一风格连续生成 5 次，每组至少 1 个和弦不同

### US-3 (P1)：和弦试听

**As a** 即兴弹琴但不确定走向是否好听的爱好者
**I want to** 听到和弦的实际听感
**so that** 不用拿起乐器就能判断这套和声的氛围

- **播放进行**：按当前速度依次弹出每个和弦的音，当前播放和弦卡片高亮
- **速度调节**：拖动速度滑块后下次循环生效
- **单和弦试听**：点击单个和弦卡片弹出该和弦的琶音

### US-4 (P2)：吉他指法参考

**As a** 吉他弹唱爱好者
**I want to** 看到和弦在吉他指板上的按法
**so that** 不用去查和弦图就能直接弹

- **查看指法**：点击和弦卡片展开 SVG 弦图 + TAB 文本
- **高品和弦**：高品位和弦显示偏移后的基准品位

### US-5 (P2)：创作笔记

**As a** 正在写歌的创作者
**I want to** 在听到和弦时立刻记下灵感的歌词片段
**so that** 灵感不会流失

- **记录笔记**：展开创作面板，输入文本，自动存盘

### US-6 (P2)：哼唱录音

**As a** 身边没有乐器但哼出旋律的创作者
**I want to** 把哼唱片段录下来绑定到当前和弦进行
**so that** 下次再打开时能回忆起当时的感受

- **录制**：授权麦克风后点击录音按钮，开始录音并显示计时
- **保存与回放**：录音出现在列表中，可播放、下载、删除
- **麦克风释放**：录音停止后硬件流已释放

### US-7 (P3)：收藏夹

**As a** 积累了多组喜欢和弦进行的用户
**I want to** 收藏、管理和快速载入这些进展
**so that** 创作素材不会丢失

- **收藏**：点击心形图标加入收藏
- **收藏夹管理**：打开收藏夹抽屉，展开笔记/录音，取消收藏
- **载入使用**：点击「载入使用」切换到该进行

### US-8 (P0)：纯节拍器

**As a** 拿起吉他想跟着节拍练琴的用户
**I want to** 打开节拍器，调节 BPM 和拍号
**so that** 不用另外开节拍器 App

- **打开节拍器**：开启节拍器开关后播放 click 声，强拍区分
- **调节 BPM**：拖动 BPM 滑块实时变化，click 间隔相应调整
- **切换拍号**：强拍间隔随拍号变化

### US-9 (P0)：节奏型试听

**As a** 想练特定扫弦节奏的吉他爱好者
**I want to** 选一套扫弦节奏型并听到方向感清晰的扫弦
**so that** 理解这个节奏型的手感和听感

- **选择节奏型**：开启节拍器 + 节奏型，当下拉选择节奏型时预览网格更新，播放时按方向扫弦
- **切换节奏型**：切换后下次小节开始时生效
- **拍号联动**：切换拍号后节奏型列表过滤为仅匹配该拍号的 pattern

### US-10 (P0)：完整试听

**As a** 听到一组和弦进行但想知道实际弹奏律动的用户
**I want to** 在节奏型下听到和弦进行
**so that** 感受到有节奏感的完整伴奏

- **完整试听**：三开关全开，先播放预备拍，之后按节奏型扫出和弦
- **回退 V2.0**：关闭节奏型开关后恢复匀速琶音模式

### US-11 (P1)：和弦按小节切换

**As a** 试听和弦进行的用户
**I want to** 每个和弦在完整小节边界上切换
**so that** 切换时机自然对齐节拍

- **小节切换**：下一小节第 1 拍切换到下一个和弦
- **循环播放**：最后一个和弦播完自动循环到第一个

### US-12 (P1)：导出收藏

**As a** 收藏了多组和弦进行的用户
**I want to** 从收藏列表中选中一些进行导出为 .zip 文件
**so that** 在另一台设备上恢复或备份数据

- **管理模式选择**：点击导出按钮后列表进入管理模式，出现复选框和全选按钮
- **打包下载**：勾选后导出，zip 内每个进行独立文件夹，含 chords.json、chords.md、recordings/

### US-13 (P1)：导入收藏

**As a** 导出过收藏想在新浏览器恢复的用户
**I want to** 选择 .zip 文件后看到预览、确认后导入
**so that** 数据完整恢复到本地

- **导入预览**：上传 zip 后预览弹窗显示进行列表，有冲突的标注并提供选项
- **执行导入**：确认后收藏、笔记、录音全部恢复，显示完成摘要

### US-14 (P2)：切换音色

**As a** 想听到更真实吉他音色的用户
**I want to** 在播放栏切换电子音色和钢弦吉他音色
**so that** 试听时有更真实的音色选择

- **切换至吉他**：点击切换按钮，下次播放时使用钢弦吉他音色
- **切换回电子**：再次点击恢复电子音色

### US-15 (P0)：和弦发声验证

**As a** 依赖和弦正确发声的开发者/用户
**I want to** 系统性验证所有和弦品质和调性下的发声正确性
**so that** 音高基座准确

- **验证和弦品质**：13 种和弦品质的发声音高与乐理理论一致
- **验证移调**：全部 12 个调下的和弦名拼写和发声音高正确
- **验证音域折叠**：极高/极低音符折叠后在合理范围内，倒置感不明显

## 10. 假设与风险

### 9.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|------|---------|-----------|
| 和弦数据本身能激发灵感 | 找目标用户试用，问「会再打开吗」 | 全部人工筛选 |
| 随机性能带来惊喜感 | 观察用户连击 ≥5 次占比 | 加过滤规则降低重复率 |
| 试听功能能让用户更快决定 | 观察试听使用率（目标 ≥40%） | 优化合成器音色 |
| 绑定笔记+录音能形成创作闭环 | 观察笔记使用率（目标 ≥15%） | 简化交互路径 |
| 用户愿意用收藏功能管理素材 | 观察收藏使用率 | 添加基础管理能力 |
| 节拍器+节奏型能提升试听体验 | 观察节奏型使用率（目标 ≥30%） | 简化交互 |
| 用户有跨设备迁移收藏的需求 | 观察导出/导入使用率 | 降低导出操作门槛 |

### 9.2 已知风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 和弦数据质量不够 | 中 | 高 | 全部人工筛选，每种风格 ≥15 组 |
| 随机重复率偏高 | 中 | 中 | 防连续重复逻辑 |
| Web Audio 在部分 Android 浏览器有兼容问题 | 中 | 中 | 惰性初始化 + 错误降级 |
| IndexedDB 在 Safari 隐私模式下不可用 | 低 | 中 | try-catch 降级，优雅提示 |
| 录音文件较大导致 zip 包体积过大 | 中 | 低 | 不做体积限制，UI 给出预计大小 |
| IndexedDB 空间不足导致导入失败 | 低 | 中 | 导入前检查剩余配额 |
