# 版本路线

> 项目版本规划，跨版本参考文档。

## 版本号规则

采用三段式语义化版本号 `X.Y.Z`，详见 `~/.claude/rules/versioning.md`。

**重大节点**：X.0 版本（如 1.0、2.0、3.0），代表产品形态或技术基座的质变。重大节点之间不做详细的 Y 版本排期，待进入该 X 版本后再规划。

---

## 重大节点

| 版本 | 状态 | 一句话 |
|------|------|--------|
| **V1.0** | 已发布 | 纯 HTML/CSS/JS 静态页，随机和弦生成 + 12 调转调 |

## 2.x 功能池

2.x 系列已按计划拆分为独立版本：

| 版本 | 状态 | 范围 | 文档 |
|------|------|------|------|
| **V2.1** | 开发中 | 节奏型系统：节拍器 + 30+ 套扫弦 pattern + 三开关自由组合 + spessasynth_lib+SF2 真人吉他音色 + 2 小节节奏型 + 两种和弦切换模式 | [V2.1-PRD](./V2.1-PRD.md) |
| **V2.2** | 规划中 | 导出/导入（jsZip .zip 打包）+ 合成器/吉他音色切换 | [V2.2-PRD](./V2.2-PRD.md) |

### V2.1 — 节奏型系统

- **节拍器底座**：BPM 40–200 可调，木鱼声（白噪声脉冲），强拍区分，支持 4/4、3/4、6/8 拍号
- **扫弦 pattern**：底层时值网格存储节奏，UI 展示方向箭头（↓↑），支持 1 小节和 2 小节（barCount 字段）
- **吉他真实音色**：spessasynth_lib + SoundFont 真人钢弦采样（Guitar_Acoustic_Stripped.sf2，34MB），扫弦和和弦播放均使用真实吉他音色
- **三开关自由组合**：节拍器/节奏型/和弦独立开关，任意组合互不锁定，三个音频层独立触发
- **内置库**：30+ 套扫弦节奏型（4/4 单小节 + 两小节 + 3/4 + 6/8 三连音）
- **两种和弦切换模式**：跟 pattern（切换周期 = 节奏型 barCount）/ 按小节
- 节拍器不依赖 SF2 加载，即时可用
- **远期**：自定义节奏型编辑器

### V2.2 — 导出/导入 + 和弦块吉他音色

- **导出**：收藏列表多选 → 打包 .zip
- zip 结构：每个进行一个文件夹，内含 `chords.json` + `chords.md` + `recordings/`
- **导入**：选 .zip → 预览 → 冲突处理（覆盖/跳过/保留两份）→ 恢复收藏、笔记、录音到浏览器存储
- 使用 jsZip 库，纯前端实现
- **音色**：播放栏电子 ↔ 钢弦吉他（Karplus-Strong）双态切换（和弦块播放用，扫弦音色已在 V2.1）

### 候选功能（待讨论确认）

- 自定义节奏型编辑器（可能放到较晚的 2.Y）
- 更多和弦进行数据扩展
- iPad/桌面端响应式适配
- 更多乐器音色选项

## 演进逻辑

```
Web 功能迭代 → PWA 安装体验 → 数据同步后端 → 跨平台 App
  (V2.1/V2.2)       (V3.0)          (V4.0)         (V5.0)
```

核心原则：**重大节点升得慢**，大部分功能演进在同一个 X 版本内完成。只有产品形态发生质变时才跳下一个重大节点。
