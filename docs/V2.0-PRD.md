## 1. 产品概述

### 1.1 一句话定位

给「略懂乐理但卡在套路的爱好者」一个不仅能看、还能听和记的完整创作工作台。

### 1.2 谁需要它

与 v1.0 保持一致，目标用户未变：

- **专业创作者** → 常见和弦走向烂熟于心，不需要工具提醒
- **完全不懂乐理的小白** → 连「和弦走向」是什么都不知道，没有这个需求
- **中间层——我们的用户** → 学过一点乐理，知道和弦走向的概念，听过 1-6-4-5、卡农进行，但对更丰富的走向没有积累。想创作但不知道从哪下手，拿起琴脑子一片空白

**v2.0 新增洞察：** 这类用户在弹到一套喜欢的和弦时，往往有「这段感觉不错，先录下来」「这个和弦配置想备注一下」的需求。v1.0 无法满足，用户只能截图或拿手机录音，心流断裂。

### 1.3 核心价值

打开网页 → 选调性 → 点一下 → 看到一组随机和弦进行 → **还能：试听音色、看吉他指法、写下歌词灵感、用哼唱记录旋律片段、收藏管理喜欢的进展**。每生成一次，不仅是看一个素材，还能马上开始创作。

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 生成成功率 | ≥90% | 生成按钮点击 / 页面打开 |
| 重复使用率 | ≥50% | 同一会话内连续生成 ≥2 次的用户占比 |
| 试听使用率 | ≥40% | 生成后点击播放按钮的用户占比 |
| 笔记使用率 | ≥15% | 生成后打开笔记面板的用户占比 |
| 操作步数 | 核心流程 ≤3 步 | 逐条验收 |

### 1.5 不做什么

不登录、不导出 PDF/图片、不教乐理课程、不集成 AI 生成、不接入后端服务。

## 2. 功能范围

### 2.1 In / Out

| 维度 | V2.0 做 | V2.0 不做 |
|------|---------|-----------|
| 调性 | 12 调选择器，默认 C 调，切换即实时映射 | 自动检测调性 |
| 风格 | 5 种：流行 / 民谣 / 摇滚 / 爵士 / R&B，各 ≥15 组 | 自定义风格、混合风格 |

| 试听 | Web Audio 三角波合成器，800Hz 低通 Rhodes 音色，循环播放，速度可调 | 采样音色、多乐器选择 |
| 指法 | SVG 吉他 6 弦 5 品弦图，横按自适应高品偏移 | 钢琴指法、尤克里里指法 |
| 笔记 | 每个进展绑随笔文本框，LocalStorage 自动存盘 | 富文本编辑、云端同步 |
| 录音 | MediaRecorder → IndexedDB，播放/下载/删除，物理麦克风释放 | 录音编辑、波形可视化 |
| 收藏 | 收藏夹侧滑抽屉，行内笔记编辑器 + 录音管理器，可载入使用 | 收藏夹排序、搜索、分类 |
| 乐理帮助 | 内嵌说明弹窗（级数概念、双记谱法、转调原理） | 交互式乐理教程 |
| 技术 | Vite + React 19 + TypeScript + TailwindCSS v4 + Framer Motion | 后端、数据库、API |
| 设备 | 移动端竖屏（375-440px） | iPad / 桌面端 |
| 账号 | 无 | 登录、注册、用户系统 |
| 保存 | LocalStorage + IndexedDB 本地持久化，无云同步 | 跨设备同步 |

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 随机生成 + 调性 + 级数展示 + 记谱模式切换 | 没有核心生成就没有产品 |
| P1 | 5 种风格 × 各 ≥15 组数据 | 数据不够用户用一次就腻 |

| P3 | 乐理说明弹窗 | 降低入门门槛 |

## 3. 功能详情

### 3.1 调性与风格选择器

- 顶部并排双下拉选择器，毛玻璃圆角胶囊样式
- **调性**：12 调（C / Db / D / Eb / E / F / F# / G / Ab / A / Bb / B），默认 C
- 调性使用衬线体（Noto Serif SC），契合乐理符号的古典感
- **风格**：5 种（流行 / 民谣 / 摇滚 / 爵士 / R&B），默认流行
- 切换调性后已有和弦即时映射到新调，不重新生成
- 切换风格后立即停止当前播放

### 3.2 和弦生成（「给我灵感」按钮）

- 点击后根据当前调性和风格，随机抽取一组和弦进行
- 默认首次打开时预渲染一组（pop_classic_heart），无空状态
- 防连续重复：确保与上一次不完全相同（只有 1 组时除外）
- 纯数组抽选，零耗时，无 loading 态

### 3.3 和弦卡片区

- 阶梯式 spring 动画由左至右滑入，每张延迟 0.06s
- 每张卡片三列布局：和弦名（左，32px 衬线体）｜ 构成音（中，小字居中对齐）｜ 级数（右，17px 衬线体）
- 点击卡片 = 触发单音试听（琶音弹出 1.4s）+ 展开/收起吉他指法图
- 播放过程中的当前和弦高亮（背景色 + 缩放 + 绿色高亮环）

### 3.4 记谱模式切换

- 「记谱」切换按钮位于播放控制栏行内
- 纳什维尔数字（1-5-6-4）↔ 罗马记号（I-V-vi-IV）
- 切换即时生效，不重新生成

### 3.5 Web Audio 试听系统

- **合成器引擎**：Triangle 波波形，Biquad Lowpass 800Hz 滤波器 + Q=1.0 共振，模拟 Rhodes 电钢琴木质暖感
- **复音架构**：多 Oscillator 并联 → 共享 GainNode → 共享 BiquadFilter → AudioContext Destination
- **ADSR 包络**：Attack 0.1s linear ramp → Sustain 保持 → Release 0.5s exponential ramp 至 0.0001，消除 click 爆音
- **防爆音**：总音量 0.15 除以复音数均分，防止 clipping
- **琶音延迟**：每个和弦内音延迟 0.06s × 索引，模拟手指错落扫弦
- **MIDI 区间锁定**：自动调整到 36-80 区间，保证听感舒适
- **循环播放**：playProgression API 使用 setInterval 以 chordDuration 为间隔循环（回到索引 0）
- **速度滑块**：范围 1.2-2.6 秒 / 级，步进 0.2s
- **播放/停止按钮**：播放时显示「停止试听」（红色），停止时显示「循环播放」（绿色）

### 3.6 SVG 吉他指法图

- 点击和弦卡片展开，弹簧动画展开/收起
- **弦图规格**：6 根纵弦线（低音 E 弦加粗），5 根横品线，标品间距 16px
- **顶部标识**：开放弦（O 空心圆）和闷音弦（X 红色叉号）
- **按指圆点**：4.2px 实心深绿圆点，半透明描边
- **横按自适应**：当最低品位 > 1 且最高品位 > 4 时，锁定基准品位（baseFret）并偏移显示，左侧标注 `{baseFret}fr`
- 品线在 baseFret === 1 时加粗（模拟弦枕）
- 右侧同时显示 TAB 文本（如 `0-2-2-0-0-0`）和构成音名
- 指法数据覆盖所有 12 调 × 常见品质，约 150 组，配增强同音异名回退和 Em 安全兜底
- 使用 `useMemo` 保护高德 SVG 坐标计算，仅在 chordName 变化时重新渲染

### 3.7 创作面板

- 点击进展信息卡（显示公式名如「1-5-6-4 进行」）展开双标签面板
- 面板弹簧动画展开，内含：
  - **标签切换器**：双按钮胶囊切换「随手笔记」/「哼唱小样」，选中项带 layoutId 滑动指示器动画
  - **随手笔记**：
    - textarea 绑定当前进行 ID，输入即自动存储到 allNotes 字典（JSON → LocalStorage）
    - 右上角显示「已自动存盘」脉冲圆点状态
    - placeholder 引导用户写下歌词联想、心流随笔
  - **哼唱小样**：
    - 圆形麦克风按钮，点击开始/停止录音
    - 录音时显示实时计时（MM:SS）+ 动画波形条
    - 录制完成后自动保存 Blob → IndexedDB + 元数据 → LocalStorage
    - 录音列表：每条显示标题 + 调性标签 + 时长 + 日期
    - 每项操作：播放（Audio 播放 Blob URL）/ 下载（a 标签触发）/ 删除
    - 物理安全：stop 时 `stream.getTracks().forEach(track => track.stop())` 释放麦克风

### 3.8 收藏夹工作台

- **入口**：顶部 header 右侧文件夹心形图标
- **打开方式**：从底部弹出的侧滑抽屉（spring 弹力动画），max-height 85vh
- **布局**：含标题栏（「收藏夹」+ 关闭按钮）、统计栏（已收藏 N 个进行）、可滚动内容列表
- **每项内容**：
  - 拖动把手状标题栏显示级数公式公式（纳什维尔或罗马）+ 风格标签
  - 当前已载入的进行标注「已载入」（绿色勾 + 绿色边框）
- **行内操作按钮**：
  - 「随笔手记」— 展开行内 textarea（同创作面板风格）
  - 「哼唱小样」— 展开行内录音管理器，显示该进行下的录音列表及其播放/下载/删除
  - 「载入使用」— 将该项设置为主面板当前进行，关闭收藏夹
  - 右侧垃圾桶图标 — 取消收藏
- **空状态**：无收藏时显示提示文案 + 心形图标
- 收藏列表持久化到 LocalStorage

### 3.9 「再换一组」按钮

- 主面板底部，样式为浅色透明按钮
- 保持当前调性和风格，重新随机生成一组
- 与「给我灵感」按钮行为一致

### 3.10 乐理说明弹窗

- 顶部 header 右侧问号图标触发
- 居中弹出模态框（spring 缩放动画），背景半透明模糊遮罩
- 内容（使用 `description` 中的心形图标为分割装饰）：
  - 什么是和弦级数
  - 纳什维尔数字与罗马记号的区别
  - 无摩擦智能转调原理
- 底部「我知道了」按钮关闭

### 3.11 底部品牌标识

- 页面最底部浅色居中文字 `V2.0`

## 4. 验收标准

### US-1 (P0)：随机生成和弦进行

**As a** 会弹点吉他/键盘但不知道弹什么的爱好者
**I want to** 选个调性和风格，获得一组随机的和弦进行
**so that** 能立刻开始弹，不打断创作心流

- **Scenario 首次使用**
  - Given 打开网页
  - Then 默认调性 C、风格流行，页面已预渲染一组和弦进行（无空状态）
  
- **Scenario 连续生成不重复**
  - Given 已生成过一次
  - When 点击「给我灵感」
  - Then 新一组和弦进行，与上一次不完全相同

- **Scenario 调性实时映射**
  - Given 当前显示 C 调和弦进行
  - When 调性切换到 G
  - Then 同组级数自动映射，和弦名和构成音变为 G 调

### US-2 (P1)：风格差异化

**As a** 写了几首歌都觉得一个味的唱作人
**I want to** 在不同风格间切换并得到不同的和弦类型
**so that** 接触不熟悉的进行，写出更多样的作品

- **Scenario 风格差异**
  - Given 已生成过流行风格
  - When 切换到「爵士」并生成
  - Then 结果包含爵士特征和弦（七/九和弦），与流行明显不同

- **Scenario 同风格多样性**
  - Given 连续在「民谣」风格下生成 5 次
  - Then 每组至少 1 个和弦不同

### US-3 (P1)：和弦试听

**As a** 即兴弹琴但不确定走向是否好听的爱好者
**I want to** 听到和弦的实际听感
**so that** 不用拿起乐器就能判断这套和声的氛围

- **Scenario 播放进行**
  - Given 已生成一组和弦
  - When 点击「循环播放」
  - Then 按当前速度依次弹出每个和弦的音，当前播放的和弦卡片高亮

- **Scenario 速度调节**
  - Given 正在播放
  - When 拖动速度滑块
  - Then 下次循环使用时生效

- **Scenario 单和弦试听**
  - Given 和弦卡片已展示
  - When 点击单个和弦卡片
  - Then 弹出该和弦的琶音，持续约 1.4s

### US-4 (P2)：吉他指法参考

**As a** 吉他弹唱爱好者
**I want to** 看到和弦在吉他指板上的按法
**so that** 不用去查和弦图就能直接弹

- **Scenario 查看指法**
  - Given 和弦卡片已展示
  - When 点击卡片
  - Then 展开 SVG 吉他弦图 + TAB 文本

- **Scenario 高品和弦**
  - Given 一个包含 7 品以上按法的和弦（如 Bm7）
  - When 点击展开
  - Then 弦图显示 offset 后的基准品位并标注 `{baseFret}fr`

### US-5 (P2)：创作笔记

**As a** 正在写歌的创作者
**I want to** 在听到和弦时立刻记下灵感的歌词片段
**so that** 灵感不会流失

- **Scenario 记录笔记**
  - Given 已生成一组和弦
  - When 展开创作面板 → 切换到「随手笔记」
  - Then 输入文本，输入后 400ms 自动存入 LocalStorage，显示「已自动存盘」

### US-6 (P2)：哼唱录音

**As a** 身边没有乐器但哼出旋律的创作者
**I want to** 把哼唱片段录下来绑定到当前和弦进行
**so that** 下次再打开时能回忆起当时的感受

- **Scenario 录制**
  - Given 已授权麦克风权限
  - When 展开创作面板 → 切换到「哼唱小样」→ 点击麦克风按钮
  - Then 开始录音，显示计时器和波形动画

- **Scenario 保存与回放**
  - Given 已完成一次录音
  - Then 录音出现在列表中，可播放、下载、删除

- **Scenario 麦克风释放**
  - Given 录音停止后
  - Then 浏览器地址栏麦克风指示灯熄灭，硬件流已释放

### US-7 (P3)：收藏夹

**As a** 积累了多组喜欢和弦进行的用户
**I want to** 收藏、管理和快速载入这些进展
**so that** 创作素材不会丢失

- **Scenario 收藏**
  - Given 已生成一组和弦
  - When 点击心形图标
  - Then 该进行加入收藏夹，图标变红填充

- **Scenario 收藏夹管理**
  - Given 收藏了多个进行
  - When 打开收藏夹抽屉
  - Then 显示所有收藏，可展开笔记/录音，可取消收藏

- **Scenario 载入使用**
  - Given 收藏夹中有其他进行
  - When 点击「载入使用」
  - Then 主面板切换为该进行，收藏夹关闭

## 5. 技术设计

### 5.1 技术栈

| 层级 | 选型 | 版本 |
|------|------|------|
| 构建 | Vite | v6 |
| 框架 | React | 19 |
| 语言 | TypeScript | ~5.8.x |
| 样式 | Tailwind CSS | v4 |
| 动画 | Framer Motion (motion) | 12.x |
| 图标 | lucide-react | 最新版 |
| 字体 | Noto Serif SC / Inter / JetBrains Mono | Google Fonts |

### 5.2 数据模型

参见 [src/types.ts](src/types.ts) 核心类型定义：

```
ChordQuality   — 13 种和弦品质（Major, m, 7, maj7, m7, m9, dim, dim7, m7b5, 7b9, 13 等）
MusicStyle     — 5 种风格枚举
ChordTemplate  — 和弦模板（rootOffset + quality + 双记谱标记）
ProgressionTemplate — 进行模板（id + style + chords[] + description + exampleSongs[]）
TransposedChord — 转调后的展示和弦（和弦名 + 构成音 + MIDI 音符 + 频率）
KeyConfig      — 调性配置（name + semitones + preferFlat）
AudioMemoMeta  — 录音元数据（id + title + date + duration + key + style + progression）
```

### 5.3 和弦转调引擎

参见 [src/utils.ts](src/utils.ts)：

- 半音阶常数数组（升号/降号双版本）
- 13 种和弦品质的音程偏移表
- `transposeChord()` — 根音偏移 → 半音索引 → 智能升降号拼写（根据调性 preferFlat + 级数升降标志）
- 构成音拼写：和弦根音为降号时统一用降号表，避免同和弦内混用 D#/Eb
- MIDI 音符映射：根音固定到 Octave 3 (MIDI 48-59)，扩展音自然叠加
- 级数字符串中的 `b` 替换为 `♭` 显示

### 5.4 音频合成

参见 [src/audio.ts](src/audio.ts)：

- `AudioSynth` 单例类
- 惰性初始化 AudioContext（兼容浏览器自动播放策略）
- MIDI → 频率：标准公式 `440 * 2^((note-69)/12)`
- 和弦音域自动修正到 36-80
- `stopAll()`：清除 interval + 遍历断开活跃节点
- 多实例安全：setTimeout 延迟清理 activeNodes 引用

### 5.5 本地存储架构

| 数据类型 | 存储方式 | Key 命名 |
|---------|---------|---------|
| 收藏列表 | LocalStorage | `and_xian_favorite_progressions` |
| 笔记字典 | LocalStorage | `and_xian_progression_notes` |
| 录音元数据列表 | LocalStorage | `and_xian_audio_memos` |
| 录音音频 Blob | IndexedDB (store: audio_memos) | `rec_{timestamp}` |

IndexedDB 数据库名：`and-xian-inspiration-db`，ObjectStore 名：`audio_memos`，keyPath: `id`

### 5.6 吉他指法查找

参见 [src/guitarChords.ts](src/guitarChords.ts)：

- 约 150 组硬编码指位数据，覆盖 12 调 × 各品质
- 查找流程：精确匹配 → 同音异名回退（C#↔Db 等）→ Em 安全兜底
- BaseFret 计算：取按品最小值和最大值，当 `maxFret > 4 && minFret > 1` 时偏移

### 5.7 样式与动画设计

- **配色系统**：雾霾灰绿底色 `#e8ece4`，棉麻宣纸卡片 `#fafdf6`，老墨水文字 `#1e2618`，树荫深绿强调 `#4f6d3a`，树皮灰绿辅助 `#6b7862`
- **中文非衬线主权**：除标题「和弦灵感」和调性选择器外，所有汉字使用无衬线体

  - 点击反馈：`whileTap: scale 0.96, stiffness: 400, damping: 16`
  - 面板展开：`stiffness: 220, damping: 18`

## 7. 假设与风险

### 7.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|------|---------|-----------|
| 试听功能能让用户更快决定「这套进行喜不喜欢」 | 观察试听使用率（目标 ≥40%） | 优化合成器音色或加入更多波形选择 |
| 绑定笔记+录音能形成创作闭环 | 观察笔记使用率（目标 ≥15%） | 简化交互路径，减少开启步骤 |
| 用户愿意用收藏功能管理素材 | 观察收藏使用率 | 添加分组/排序等基础管理能力 |

### 7.2 已知风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Web Audio 在部分 Android 浏览器有兼容问题 | 中 | 中 | 惰性初始化 + 错误降级，不影响页面其他功能 |
| IndexedDB 在 Safari 隐私模式下不可用 | 低 | 中 | try-catch 降级，录音功能优雅提示 |
| 收藏/笔记数据丢失（LocalStorage 清空） | 低 | 低 | 纯本地存储，用户预期明确；考虑后续导出功能 |
