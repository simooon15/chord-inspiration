## 1. 产品概述

### 1.1 更新背景

V1.0 在 `max-width: 414px` 固定宽度下开发和测试。上线后用户在 iPhone 17（CSS 宽度 402px）及其他主流机型上发现两个问题：

- **横向不适配**：414px 固定容器在 402px 屏宽下表现异常
- **纵向过长**：页面信息可在一屏展示，但固定间距和 padding 导致需要滚动

### 1.2 更新目标

页面在主流 iPhone CSS 宽度范围（375-440px）内自适应，核心信息在竖屏下一屏可见。

### 1.3 影响范围

| 机型 | CSS 宽度 | V1.0 表现 | V1.0.1 目标 |
|------|---------|----------|-----------|
| iPhone SE | 375px | 可行，卡片窄 | 自适应 |
| iPhone 15/16 | 390px | 基本适配 | 自适应 |
| **iPhone 17** | **402px** | **不适配（主要问题）** | 自适应 |
| iPhone 17 Air | 420px | 待验证 | 自适应 |
| iPhone 17 Pro Max | 440px | 容器太窄 | 自适应 |

## 2. 功能范围

### 2.1 V1.0.1 In / Out

| 维度 | V1.0.1 做 | V1.0.1 不做 |
|---|---|---|
| 容器宽度 | `max-width: 100%`，取消固定 px 值 | iPad 适配 |
| 横向内边距 | 百分比或 viewport 相对单位 | 桌面端适配 |
| 纵向间距 | 缩小顶部留白和区块间距，减少总高度 | 收藏功能 |
| 字号 | 可选：根据 viewport 微调 | 和弦数据更新 |
| 触摸交互 | 保持 V1.0 动效体系不变 | 新增控件 |
| 视觉风格 | 保持 Liquid Glass 森林绿不变 | 配色方案修改 |

## 3. 功能详情

### 3.1 容器自适应

**当前**：`max-width: 414px` + `padding: 72px 24px ...`
**目标**：宽度随屏幕自适应，横向下限 375px、上限 440px

### 3.2 纵向压缩

**当前页面构成**（估算高度，375px 屏宽下）：

| 元素 | 当前高度(约) | 可压缩？ |
|------|-----------|---------|
| 顶部留白 | 72px | 可缩至 48-60px |
| 标题行 | ~50px | 保持 |
| 控件行 | ~46px | 保持 |
| 控件→按钮间距 | 20px | 可缩至 16px |
| 给我灵感按钮 | ~48px | 保持 |
| 按钮→和弦区 | 36px | 可缩至 24-28px |
| 和弦行 ×4 + 间距 | ~264px | 保持 |
| 和弦区→刷新 | 36px | 可缩至 24-28px |
| 刷新按钮 | ~48px | 保持 |
| 底部留白 | 48px | 可缩至 32px |

**压缩目标**：总高度减少 40-60px，确保 iPhone SE (375×667) 也能一屏看完核心信息。

## 4. 验收标准

### Scenario: iPhone 17 竖屏打开
- **Given:** 我用 iPhone 17（402×874 CSS px）打开网页
- **When:** 页面加载完成
- **Then:** 所有元素横向适配屏幕，无溢出；标题、控件、和弦卡片、刷新按钮无需滚动即可看完

### Scenario: iPhone SE 竖屏打开（下限测试）
- **Given:** 我用 iPhone SE（375×667 CSS px）打开网页
- **When:** 页面加载完成
- **Then:** 核心信息（标题→刷新按钮）在竖屏一屏内可见

### Scenario: iPhone 17 Pro Max 竖屏打开（上限测试）
- **Given:** 我用 iPhone 17 Pro Max（440×956 CSS px）打开网页
- **When:** 页面加载完成
- **Then:** 内容居中，不因屏幕变宽而散开
