## 2. 节奏型数据层

### 2.1 类型定义 (`src/rhythm/types.ts`)

```typescript
export type TimeSignature = '4/4' | '3/4' | '6/8';
export type StrumDirection = 'down' | 'up';
export type StrumSlot = StrumDirection | null;

export interface BeatGrid {
  subdivision: number;        // 每拍子分格数，4 = 十六分音符
  slots: StrumSlot[];         // 长度 === subdivision，每个格子的扫弦方向
}

export interface RhythmPattern {
  id: string;                 // 唯一标识，如 "folk_basic"
  name: string;               // 中文名，如 "民谣基础扫弦"
  timeSignature: TimeSignature;
  beats: BeatGrid[];          // 长度由拍号决定：4/4→4, 3/4→3, 6/8→6
}
```

## 3. 播放调度引擎 (`src/rhythm/engine.ts`)

### 3.1 职责

替换 V2.0 的 `setInterval(chordDuration)` 简单循环，升级为以 BPM 为时钟源、按格子 tick 推进的调度器。

### 3.2 核心接口

```typescript
export class RhythmEngine {
  private bpm: number;
  private timeSignature: TimeSignature;
  private pattern: RhythmPattern | null;
  private metronomeOn: boolean;
  private rhythmOn: boolean;
  private chordsOn: boolean;
  private tickIntervalId: number | null;
  private currentBeat: number;     // 当前拍号索引 (0-based)
  private currentSlot: number;     // 当前格子索引 (0-based)
  private currentChordIndex: number;
  private tickCount: number;       // 总 tick 数（用于预备拍计数）

  // 回调
  onTick?: (info: TickInfo) => void;       // 每个 tick 触发
  onChordChange?: (index: number) => void; // 和弦切换时触发
  onCountInEnd?: () => void;               // 预备拍结束
}
```

### 3.3 Tick 时钟

```
tickInterval = (60 / BPM / subdivision) × 1000 ms
```

- 4/4, subdivision=4: interval = 60000 / BPM / 4 ms
- BPM=100 时，interval = 150ms（每个十六分音符 150ms）
- BPM=200 时，interval = 75ms
- BPM=40 时，interval = 375ms

**时钟精度校准：** 使用 `performance.now()` 记录起始时间戳，每 tick 计算实际经过时间与理论时间的偏差，累计超过半个 tick 时补跳一个 tick。

```typescript
private scheduleNextTick(): void {
  const expectedTickTime = this.tickCount * this.tickIntervalMs;
  const actualElapsed = performance.now() - this.startTime;
  const drift = actualElapsed - expectedTickTime;

  // 漂移补偿：跳过或延迟
  const adjustedDelay = Math.max(0, this.tickIntervalMs - drift);
  this.tickTimerId = window.setTimeout(() => this.fireTick(), adjustedDelay);
}
```

### 3.4 Tick 事件处理

每个 tick 依次执行：

```
fireTick()
  ├─ 预备拍阶段 (tickCount < ticksPerBar)：
  │    └─ 节拍器 ON → 播放 click（强拍/弱拍）
  │
  └─ 正常播放阶段 (tickCount >= ticksPerBar)：
       ├─ slot === 0 → 节拍器 ON → 播放 click（强拍/弱拍）
       ├─ 节奏型 ON && slots[slot] !== null → 播放扫弦
       │    ├─ 和弦 ON → StrumAudio.playStrum(midiNotes, direction)
       │    └─ 和弦 OFF → StrumAudio.playMutedStrum(direction)
       ├─ 最后一格 && 和弦 ON → 推进 currentChordIndex（循环）
       └─ currentSlot++, 回卷 currentBeat
```

### 3.5 启动/停止

```typescript
start(): void {
  this.stop();
  this.startTime = performance.now();
  this.tickCount = 0;
  this.currentBeat = 0;
  this.currentSlot = 0;
  this.currentChordIndex = 0;
  this.scheduleNextTick();
}

stop(): void {
  if (this.tickTimerId) clearTimeout(this.tickTimerId);
  // 停止所有活跃的扫弦节点
  this.strumAudio.stopAll();
}
```

## 4. 音频层改动 (`src/audio.ts`)

### 4.1 节拍器音色 (`MetronomeSound`)

在 `AudioSynth` 类中新增方法：

```typescript
playClick(isStrongBeat: boolean, volumeMultiplier = 1.0): void {
  const ctx = this.getContext();
  const now = ctx.currentTime;

  const freq = isStrongBeat ? 1200 : 800;
  const peakVolume = (isStrongBeat ? 0.3 : 0.2) * volumeMultiplier;

  const osc = ctx.createOscillator();
  const gain = ctx.createGain();

  osc.type = 'sine';
  osc.frequency.setValueAtTime(freq, now);

  gain.gain.setValueAtTime(0, now);
  gain.gain.linearRampToValueAtTime(peakVolume, now + 0.001);  // Attack 1ms
  gain.gain.exponentialRampToValueAtTime(0.0001, now + 0.04);  // Decay 40ms

  osc.connect(gain).connect(ctx.destination);
  osc.start(now);
  osc.stop(now + 0.05);
}
```

**模式自适应音量：**
- 纯节拍器模式（和弦 OFF）：`volumeMultiplier = 1.3`
- 混合模式（和弦 ON）：`volumeMultiplier = 0.8`

### 4.2 方向感知扫弦 (`playStrum`)

在现有 `playChord()` 基础上增加方向参数：

```typescript
playStrum(midiNotes: number[], direction: StrumDirection): void {
  const ctx = this.getContext();
  const now = ctx.currentTime;

  // 按下行/上行排序
  const sortedNotes = direction === 'down'
    ? [...midiNotes].sort((a, b) => a - b)     // 低→高
    : [...midiNotes].sort((a, b) => b - a);    // 高→低

  const noteCount = sortedNotes.length;
  const strumDelay = 0.04;                       // 每音间隔 40ms
  const noteDuration = 0.6;                      // 单音持续 600ms

  const filter = ctx.createBiquadFilter();
  filter.type = 'lowpass';
  filter.frequency.setValueAtTime(800, now);
  filter.Q.setValueAtTime(1.0, now);
  filter.connect(ctx.destination);

  sortedNotes.forEach((midi, i) => {
    const note = this.clampMidi(midi);           // 复用 V2.0 的 36-80 折叠
    const freq = this.midiToFreq(note);

    const osc = ctx.createOscillator();
    const gain = ctx.createGain();
    osc.type = 'triangle';
    osc.frequency.setValueAtTime(freq, now);

    const noteStart = now + i * strumDelay;
    const maxVol = 0.12 / Math.max(3, noteCount);

    gain.gain.setValueAtTime(0, now);
    gain.gain.linearRampToValueAtTime(maxVol, noteStart + 0.05);
    gain.gain.setValueAtTime(maxVol, noteStart + noteDuration - 0.3);
    gain.gain.exponentialRampToValueAtTime(0.0001, noteStart + noteDuration);

    osc.connect(gain).connect(filter);
    osc.start(noteStart);
    osc.stop(noteStart + noteDuration + 0.1);

    this.activeNodes.push({ osc, gain });
  });
}
```

### 5.2 播放模式路由

播放按钮 `handlePlay()` 的逻辑：

```typescript
function handlePlay() {
  if (isPlaying) {
    engine.stop();
    synth.stopAll();
    setIsPlaying(false);
    return;
  }

  if (rhythmOn) {
    // 使用新的 RhythmEngine
    engine.configure({ bpm, timeSignature, pattern, metronomeOn, rhythmOn, chordsOn });
    engine.onTick = (info) => {
      setCurrentTickBeat(info.beat);
      setCurrentTickSlot(info.slot);
      setActiveChordIndex(info.chordIndex);
    };
    engine.start();
  } else {
    // 使用 V2.0 的 synth.playProgression()
    synth.playProgression(midiArrays, chordDuration, setActiveChordIndex);
  }
  setIsPlaying(true);
}
```

### 5.3 播放中切换参数

如果正在播放中修改 BPM/拍号/pattern，不中断播放，在**下一小节边界**生效：

```typescript
useEffect(() => {
  if (isPlaying && rhythmOn) {
    engine.updateConfig({ bpm, timeSignature, pattern });
  }
}, [bpm, timeSignature, selectedPatternId]);
```

`engine.updateConfig()` 内部标记参数变更，在下一个 `currentBeat === 0 && currentSlot === 0` 时应用。

### 5.4 停止时清理

```typescript
function handleStop() {
  engine.stop();
  synth.stopAll();
  setIsPlaying(false);
  setActiveChordIndex(-1);
  setCurrentTickBeat(0);
  setCurrentTickSlot(0);
}
```

## 7. 预览网格渲染

### 7.1 格子计算

```typescript
// 从选中 pattern 计算总格子数
const ticksPerBar = pattern.beats.reduce((sum, b) => sum + b.subdivision, 0);
// 4/4, sub=4:  ticksPerBar = 16
// 3/4, sub=4:  ticksPerBar = 12
// 6/8, sub=4:  ticksPerBar = 24
```

### 7.2 渲染结构

不创建独立组件文件，在 App.tsx 的播放栏区域直接渲染（与 V2.0 组件组织方式一致）。

使用 `motion.span` 渲染每个格子，当前高亮格子使用 `layoutId="rhythm_cursor"` 实现平滑过渡（可选，也可直接用 `animate` 切换 backgroundColor）。

### 7.3 格子高亮过渡

高亮格子在每 tick 切换。直接使用 `motion.span` 的 `animate` prop 切换 `backgroundColor`，spring 过渡 300ms，视觉上平滑。

## 8. 与 V2.0 的兼容策略

| 场景 | 行为 |
|------|------|
| 节奏型 OFF | 100% V2.0 行为：`playProgression()` + 和弦时长滑块 |
| 节奏型 ON，和弦 OFF | Engine tick 驱动 → `playClick()` + `playMutedStrum()` |
| 节奏型 ON，和弦 ON | Engine tick 驱动 → `playClick()` + `playStrum()` |
| 节拍器 OFF，节奏型 ON，和弦 ON | Engine tick 驱动 → 仅 `playStrum()`（无 click） |
| 播放中切换开关 | 不中断播放，下一小节边界生效 |

## 9. 和弦发声验证

V2.1 在开发前需要验证 V2.0 的和弦音频基座。验证方法：基于现有 `transposeChord()` 和 `playChord()`，编写一次性验证脚本或手动测试清单，遍历 13 种品质 × 12 调，确认 MIDI 音符与理论音高一致。

验证重点：
- `CHORD_FORMULAS` 半音偏移正确性
- `transposeChord()` 移调后拼写和 MIDI notes 正确
- `clampMidi()` 折叠边界（36-80）不产生明显倒置

此验证为一次性开发前置步骤，不产生持久化代码。
