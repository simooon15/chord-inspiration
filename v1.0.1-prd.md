## 1. 变更背景

V1.0 以 `max-width: 414px` 固定宽度开发和测试。上线后出现四个问题：

| # | 问题 | 影响 |
|---|------|------|
| 1 | 414px 固定容器在 iPhone 17（402px）和 17 Pro Max（440px）下宽度不适配 | iPhone 17 系列表现异常 |
| 2 | 固定间距和 padding 导致总高度过大 | 需要滚动才能看完全部信息 |
| 3 | 标题使用系统 sans-serif，与和弦卡片的衬线字体缺乏层次对比 | 品牌感单薄 |
| 4 | `.chord-item.animate-in` 的 `animation-fill-mode: both` 持续持有 `transform`，与 `:active` 的 `scale(0.88)` 冲突 | 点击卡顿 |

## 2. 变更范围

| 维度 | V1.0.1 做 | V1.0.1 不做 |
|------|-----------|-------------|
| 布局 | 容器宽度自适应 375-440px，内边距用 viewport 相对单位 | iPad / 桌面端适配 |
| 间距 | 缩小顶部留白和区块间距，减少总高度 | — |
| 字体 | 标题改用思源宋体 Noto Serif SC | 其他文字字体修改 |
| 动效 | 修复和弦卡片 :active 卡顿 | 新增动效 |
| 视觉 | 保持 Liquid Glass 森林绿不变 | 配色方案修改 |

### 响应式覆盖范围

| 机型 | CSS 宽度 | V1.0 | V1.0.1 |
|------|----------|------|--------|
| iPhone SE | 375px | 可行 | 自适应 |
| iPhone 15/16 | 390px | 基本适配 | 自适应 |
| iPhone 17 | 402px | 不适配 | 自适应 |
| iPhone 17 Air | 420px | 未验证 | 自适应 |
| iPhone 17 Pro Max | 440px | 容器太窄 | 自适应 |

## 3. 功能详情

> 以下仅列出相对于 v1.0 有变化的条目。无变化的条目（调性选择器、风格选择器、生成按钮等）保持不变，参见 v1.0 PRD。

### 3.1 容器自适应

- 取消 `max-width: 414px` 固定值
- 改为 `width: 100%; max-width: 440px`
- 横向内边距改用 `clamp(16px, 5vw, 24px)` 自适应

### 3.2 纵向间距压缩

具体调整：

| 区域 | V1.0 | V1.0.1 | 变化 |
|------|------|--------|------|
| 顶部留白 | 72px | 52px | -20px |
| 控件 → 按钮 | 20px | 16px | -4px |
| 按钮 → 和弦区 | 36px | 0 | -36px |
| 和弦区 → 刷新 | 36px | 0 | -36px |
| 底部留白 | 48px | `max(20px, env(safe-area-inset-bottom))` | 自适应 |

总计减少约 50-60px，iPhone SE 竖屏可一屏看完核心信息。

### 3.3 标题字体

- 标题「和弦灵感」从系统 sans-serif → Noto Serif SC（思源宋体），weight 575
- Google Fonts 子集化加载，仅包含页面所需 5 个汉字，不影响速度

### 3.4 和弦卡片点击优化

- **根因**：入场动画结束后 `animation-fill-mode: both` 让 `transform: translateX(0)` 持续占用，WebKit 合成器中 CSS animation 的 transform 优先级高于 `:active` 伪类，导致 `scale(0.88)` 冲突
- **修复**：`animationend` 事件中移除 `.animate-in` 类并清除 `animation` 属性，将 transform 交还 `:active` 独占
- 同时清理未使用的 `.chord-item.tapped` CSS 规则

## 4. 验收标准

> 以下为 v1.0.1 增量验收标准。v1.0 的验收标准仍需全部通过。

### Scenario：iPhone 17 竖屏（上限）

- Given iPhone 17 Pro Max（440×956 CSS px）
- When 打开网页
- Then 内容居中，不因屏幕变宽而散开

### Scenario：iPhone SE 竖屏（下限）

- Given iPhone SE（375×667 CSS px）
- When 打开网页
- Then 标题到刷新按钮在竖屏一屏内可见，无需滚动

### Scenario：标题字体

- Given 任意设备打开网页
- When 页面加载完成
- Then 标题「和弦灵感」以思源宋体渲染

### Scenario：和弦卡片点击流畅

- Given 和弦卡片已展示（入场动画已完成）
- When 手指快速连击卡片 3 次
- Then 每次点击即时缩放至 0.88 并弹回，无延迟、无卡顿

## 5. 技术设计

> 变更仅涉及 CSS 和少量 JS，数据层无变化。

### 5.1 关键 CSS 变更

```css
/* 容器 */
.app-container {
  width: 100%;
  max-width: 440px;  /* was 414px */
  padding: 0 clamp(16px, 5vw, 24px);
}

/* 标题 */
h1 {
  font-family: 'Noto Serif SC', serif;
  font-weight: 575;
}
```

### 5.2 关键 JS 变更

```js
item.addEventListener('animationend', function() {
  item.classList.remove('animate-in');
  item.style.animation = 'none';  // 释放 transform 给 :active
});
```
