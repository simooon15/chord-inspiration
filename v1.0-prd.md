## 1. 产品概述

### 1.1 产品背景

音乐创作者（吉他/钢琴/尤克里里爱好者、独立唱作人）在创作时经常面临「拿起乐器不知道弹什么」的困境。现有方案——翻老歌、刷 YouTube 教程、翻和弦书——流程太长，找灵感 30 分钟、真正弹 10 分钟，严重打断创作心流。

### 1.2 产品定位

**一个「灵感启动器」而非「和弦工具」。** 单页网页应用，用户选调性、选风格、点一下按钮，立刻看到一组随机和弦进行（含级数标注）。不需要登录、不需要设置、打开就能用。

### 1.3 目标用户

| 用户画像 | 特征 | 核心痛点 |
|---|---|---|
| 卧室唱作人 | 会弹吉他/钢琴，偶尔写歌，乐理基础薄弱 | 不知道从哪开始，总在同样套路里循环 |
| 即兴爱好者 | 碎片时间摸琴，追求好玩和新鲜 | 找灵感的过程太慢，打断心流 |
| 乐理学习者 | 正在学和弦，想通过实例理解进行逻辑 | 看不懂和弦之间的相对关系 |

### 1.4 产品目标

| 指标 | 目标值 | 衡量方式 |
|---|---|---|
| 核心操作完成率 | ≥90% 用户能完成一次生成 | 生成按钮点击 / 页面打开 |
| 重复使用率 | ≥50% 用户会再点一次生成 | 同一会话内连续生成次数 |
| 操作效率 | 核心流程 ≤3 步 | 按验收条件逐条验证 |

## 2. 用户洞察

### 2.1 使用场景

| 维度 | 描述 |
|---|---|
| 用户 | 弹吉他/钢琴/尤克里里的音乐爱好者、独立唱作人 |
| 情境 | 拿起乐器想写东西但脑子空白，或卡在同一套路里循环。常用手机（放琴边单手操作）或 iPad（放谱架上） |
| 当前方案 | 翻老歌找和弦、搜 YouTube 教程、查 Ultimate Guitar、瞎弹碰运气、买和弦书 |
| 核心任务 | 快速获得「能激发灵感」的和弦进行，然后开始创作 |

### 2.2 用户任务

- **功能性**：选风格 → 获得和弦进行 → 一键换新 → 了解和弦构成音
- **社交性**：弹出「听起来很厉害」的和弦显得有功底；写出不像口水歌的作品获得认可
- **情感性**：摆脱「我不会写歌」的自我怀疑；获得灵感被点燃的兴奋感；避免抱着琴半小时没产出

### 2.3 痛点

- 乐理基础薄弱，不知道怎么「造」和弦进行，凭感觉弹
- 现有工具太重：搜 YouTube → 看视频 → 暂停 → 试弹，流程打断心流
- 总是回到熟悉套路（永远 1-6-4-5），写出来的东西都一样
- 找到好听的和弦忘记录，第二天就忘了
- 设备适配缺失：手机屏幕太小，iPad 看谱工具未适配

### 2.4 收益期望

- 3 步内完成：打开 → 选风格 → 点一下 → 开始弹
- 每次结果不同，保持新鲜感；免费、不用注册、即开即用
- 创作频率变高——启动摩擦从「找 30 分钟」变成「点一下就有」

## 3. 功能范围

### 3.1 MVP In / Out

| 维度 | V1.0 做 | V1.0 不做 |
|---|---|---|
| 调性 | 顶部下拉框，12 个调，默认 C 调 | 自动检测调性 |
| 风格 | 5 种（流行/民谣/摇滚/爵士/R&B），默认流行 | 自定义风格 |
| 生成 | 选调+风格 → 随机生成一组和弦 | AI 生成、用户输入条件 |
| 显示 | 和弦名 + 构成音 + 纳什维尔数字级数 | 五线谱、指位图、音频播放 |
| 交互 | 一键换一组、换调即时映射 | 撤销、历史记录 |
| 技术 | 纯前端 HTML/CSS/JS，零依赖；数据独立 `chords.js` | 数据库、API、用户系统 |
| 设备 | 仅移动端竖屏 | iPad/桌面 |
| 保存 | 不做 | 收藏、导出、账号 |

### 3.2 版本规划

| 版本 | 内容 | 依据 |
|---|---|---|
| V1.0 MVP | P0 随机生成 + P1 多风格 + 调性/级数 + P2 极简约束 | 验证核心价值 |
| V1.0.1 | 手机响应式适配 375-440px | 适配新机型 |
| V2.0 | 收藏（localStorage）+ iPad 响应式 | 留存 + 覆盖 iPad |
| V3.0 | 和弦指位图 | 降低弹奏门槛 |
| 不做 | 音频播放 | 用户有乐器，自己弹 |

### 3.3 优先级矩阵

| 优先级 | 功能 | 用户价值 | 实现难度 | 版本 |
|---|---|---|---|---|
| P0 | 选调+风格 → 随机生成和弦+级数 | 高：解决「没起点」和「不能转调」 | 低 | V1.0 |
| P1 | 5 种风格 ×16 组，每次不重复 | 高：解决「套路循环」 | 低 | V1.0 |
| P2 | 3 步内完成核心操作 | 高：设计原则，不独立开发 | — | V1.0 |
| P3 | 收藏喜欢的进行 | 中 | 中 | V2 |
| P3.5 | iPad 响应式适配 | 中 | 低 | V2 |
| P4 | 和弦指位图 | 低-中 | 高 | V3 |
| — | 音频播放 | 低-中 | 高 | 不做 |

P2 不是独立功能，是**所有功能的验收门槛**：任何新增功能，操作步数增量 ≤1。

## 4. 功能详情

### 4.1 调性选择器

- 位置：页面顶部，固定可见
- 交互：`<select>` 下拉框，默认「C 调」，12 个调可选
- 联动：切换调性后，当前和弦即时映射到新调，无需重新生成

### 4.2 风格选择器

- 位置：调性选择器下方
- 交互：`<select>` 下拉框，默认「流行」，5 种风格可选

### 4.3 给我灵感按钮

- 点击后根据当前调性和风格，随机生成一组和弦进行
- 防连续重复：do-while 循环确保不与上一次完全相同

### 4.4 结果展示

每张和弦卡片三列布局：

```
│ 和弦名（左，大号衬线 32px）│ 构成音（中）│ 级数（右，17px）│
```

- 每列三列垂直居中：和弦名（左锚 72px）、构成音（flex:1 居中）、级数（右锚 36px）
- 入场动画：四张卡片阶梯滑入 800ms，延迟 0/60/120/180ms

### 4.5 再换一组按钮

- 保持当前调性和风格，随机生成新一组

## 5. 验收标准

### US-1 (P0): 按风格随机生成和弦进行

- **As a** 吉他/键盘爱好者，拿着乐器想写点东西但脑子一片空白
- **I want to** 选择调性、选择风格，获得一组随机的和弦进行
- **so that** 我能立刻用自己熟悉的调开始弹，看懂和弦之间的相对关系

**Scenario:** 首次使用
- **Given:** 打开网页，调性默认 C 调，风格默认流行，无需登录
- **When:** 保持默认或切换选择，点击「给我灵感」
- **Then:** 显示当前调性的一组和弦，每个和弦标注纳什维尔级数，操作 ≤3 步

**Scenario:** 切换调性即时更新
- **Given:** 当前显示 C 调流行进行
- **When:** 将调性从 C 切换到 G
- **Then:** 同组级数自动映射为 G 调和弦，无需重新生成

**Scenario:** 连续生成结果不同
- **Given:** 已经生成过一次
- **When:** 再次点击「给我灵感」
- **Then:** 显示新一组和弦进行，与上一次不完全相同

### US-2 (P1): 多种风格覆盖

- **As a** 独立唱作人，写了几首歌都感觉一个味
- **I want to** 在不同风格之间切换，每次生成结果都不同
- **so that** 我能接触不熟悉的进行，写出风格更多样的作品

**Scenario:** 风格切换
- **Given:** 当前在「流行」风格
- **When:** 切换到「爵士」并生成
- **Then:** 显示的和弦包含爵士特征（七和弦/九和弦），与流行风格明显不同

**Scenario:** 同风格不重复
- **Given:** 连续生成 5 次「民谣」
- **Then:** 每组和弦不完全相同（至少 1 个和弦不同）

### US-3 (P2): 极简操作，三步内完成

- **As a** 正在即兴弹琴的音乐爱好者
- **I want to** 在尽可能少的操作内获得一组和弦
- **so that** 注意力始终留在乐器上，不被界面打断

**Scenario:** 首次使用
- **Given:** 打开网页，调性和风格已默认选中无需操作
- **When:** 点击「给我灵感」
- **Then:** 1 步完成；即使切换选项，切换→生成仍 ≤3 步

**Scenario:** 一键刷新
- **Given:** 已经生成过一次
- **When:** 点击「再换一组」
- **Then:** 1 步完成，保持当前调性和风格

## 6. 数据源设计

和弦数据存储在独立文件 `chords.js`，5 种风格各 15-16 组，以 C 调为基准：

```
{degrees: [1,5,6,4], chords: ['C','G','Am','F'], notes: ['C-E-G','G-B-D','A-C-E','F-A-C']}
```

12 调音名映射表 + 转调函数 `transposeChord()` / `transposeNote()`，供页面实时调用。

## 8. 假设与风险

### 8.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|---|---|---|
| 和弦数据有用 | 给 3 个玩乐器的朋友试用，问「会再打开吗」 | 补数据，全部人工筛选 |
| 用户不需要播放就能判断 | 观察是否有人说「能放一下吗」 | V2 加 Web Audio 预览 |
| 随机性带来惊喜 | 观察是否连击 ≥5 次 | 加调性过滤 |

### 8.2 风险

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 和弦数据太水 | 中 | 高 | 全部人工筛选，不用 AI 生成 |
| 随机重复率高 | 中 | 中 | 每种 ≥15 组 + 去重 |
| 移动端浏览器兼容 | 低 | 中 | 只用标准 HTML/CSS |
