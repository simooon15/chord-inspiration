## 变更背景

V2.0 做了一个完整创作工作台：能听和弦、看指法、写笔记、录哼唱、管理收藏。用户打开网页就能完成从「获得灵感」到「记录灵感」的闭环。但上线后，三个新问题浮现：

**1. 试听是「平的」。** V2.0 的匀速琶音能展示和弦色彩，但真实音乐不是每个音等时长地往上弹。用户听到一组进行后，脑子里浮现的是一个有律动的伴奏，而 V2.0 给出的是一串机械的琶音——信息量不够。用户需要的不是「这四个音是什么」，而是「用这个节奏弹这四个和弦是什么感觉」。

**2. 练琴场景缺位。** V2.0 的设计场景是「浏览灵感」，但大量用户的使用场景是「拿着琴跟着弹」。他们需要一个节拍器来保持速度，需要扫弦节奏型来练习手感——但这些在 V2.0 里都没有。用户只能一边开着和弦灵感，一边另外打开节拍器 App，体验割裂。

**3. 数据困在浏览器里。** V2.0 的收藏、笔记、录音全存在 LocalStorage 和 IndexedDB 里，清缓存就没了，换设备也带不走。用户积累了几十组素材后开始焦虑——这些心血没有备份，也不敢清理浏览器数据。

**决策：三个模块并行推进。** 节奏型系统是 V2.x 的核心——它不是「加一个功能」，而是改变了试听的产品形态，从匀速琶音升级为有律动的完整伴奏。节拍器让它在练琴场景下独立可用。导出/导入解决数据安全感问题。钢弦吉他音色作为音色扩展的低成本尝试。四个模块在 V2.0 的工程基座上增量构建，不改变技术架构。

## 1. 产品概述

### 1.1 一句话定位

给创作者加上可调节的节拍器和扫弦节奏型，让和弦试听从「匀速琶音」升级为「有律动的完整伴奏」。

### 1.2 谁需要它

与 V2.0 一致，目标用户未变。核心场景：

- 用户在 V2.0 里听到一组和弦进行，觉得不错，但不知道实际弹出来是什么节奏感。匀速琶音只能听和声色彩，听不出律动。
- 用户拿起吉他想跟着练，需要一个节拍器。V2.0 没有节拍器，用户只能另外开一个节拍器 App。
- 用户想练特定风格的扫弦（民谣 ↓↓↑、摇滚强拍等），V2.0 无法提供节奏型参考。

### 1.3 核心价值

**节奏型系统：**

三个独立开关自由组合：

- **节拍器**：木鱼音色，强拍区分，BPM 40-200 可调，支持 4/4、3/4、6/8 拍号
- **节拍器 + 节奏型**：在节拍器基础上叠加扫弦方向（↓↑），练扫弦手感
- **节拍器 + 节奏型 + 和弦**：完整试听——在某个节奏型下听到和弦进行，模拟真实弹奏的律动

节奏型 ON 时，取代 V2.0 的匀速琶音模式。

**导出/导入：**

收藏的多选打包下载，含和弦数据、笔记和录音，可在浏览器间迁移。导入时冲突预览，用户决定覆盖/跳过/保留。

**钢弦吉他音色：**

在电子音色之外增加 Karplus-Strong 合成的钢弦吉他音色，播放栏一键切换，让试听更真实。

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 节奏型使用率 | ≥30% | 试听时开启节奏型的用户占比 |
| 节拍器独立使用率 | ≥15% | 仅开启节拍器（不叠加和弦）的用户占比 |
| BPM 调节率 | ≥40% | 开启后调整过 BPM 的用户占比 |
| 节奏型切换率 | ≥20% | 开启后切换过不同节奏型的用户占比 |

### 1.5 不做什么

- 不做自定义节奏型编辑器（远期）
- 不做多小节节奏型
- 不做节奏型按风格分类
- 不做采样类真实乐器音色（延续 Web Audio 合成方案）
- 不做按拍切换和弦（v1 仅按小节切换）

## 3. 功能详情

### 3.1 节拍器底座

- 通过 Web Audio API 合成的短促木鱼/鼓边音色
  - 强拍（第 1 拍）：频率稍高（约 1200Hz），响度稍大
  - 弱拍（其余拍）：频率稍低（约 800Hz），响度稍小
  - 使用正弦波 + 极短 Attack/Decay 包络模拟打击乐瞬态
- **BPM 控制**：水平滑块 + 右侧数字显示，范围 40-200，步长 1
  - 拖动滑块粗调，数字实时跟随
  - 默认 BPM：100
- **拍号选择器**：分段按钮，默认 4/4，可选 3/4、6/8
  - 切换拍号后：节拍器强拍间隔、预览网格列数、节奏型列表过滤（仅显示匹配拍号）联动变化
- **预备拍**：点击播放后先播放 1 小节纯节拍器（不叠加和弦和节奏型），之后自动进入选定模式

### 3.2 节奏型数据模型

- **拍级数组**：一个小节 = `beats[]`，每个 beat 独立定义 `subdivision`（4 = 十六分音符，3 = 三连音）和 `slots[]`（每个格子为 `'down' | 'up' | null`）
- **拍号内嵌**：每个节奏型数据自带 `timeSignature` 字段，选中时自动同步拍号选择器
- **内置库**：15-20 套节奏型，覆盖常见的 4/4、3/4 扫弦 pattern，开发时从网上吉他教学资源收集
- **数据只读**：2.x 不提供编辑功能，用户仅可浏览和选择

示例数据结构：

```
{
  id: "folk_basic",
  name: "民谣基础扫弦",
  timeSignature: "4/4",
  beats: [
    { subdivision: 4, slots: [down, null, null, null] },
    { subdivision: 4, slots: [down, null, up,   null] },
    { subdivision: 4, slots: [down, null, null, null] },
    { subdivision: 4, slots: [down, null, up,   null] },
  ]
}
```

### 3.3 三开关模式

- 三个独立开关按钮，位于节奏面板顶部
  - **节拍器**：ON → 播放 click 声 + 预备拍；OFF → 静音
  - **节奏型**：ON → 按选定 pattern 的节奏触发扫弦，取代匀速琶音；OFF → 回退到 V2.0 匀速琶音（仅当和弦开关 ON 时有效）
  - **和弦**：ON → 播放和弦进行音色；OFF → 仅节拍器 + 节奏型方向（无声扫弦或不触发音高）
- 开关状态互不锁定，自由组合
- 典型组合：
  - 节拍器 ON / 节奏型 OFF / 和弦 OFF → 纯节拍器
  - 节拍器 ON / 节奏型 ON / 和弦 OFF → 节拍器 + 可视节奏型（练扫弦）
  - 节拍器 ON / 节奏型 ON / 和弦 ON → 完整试听
  - 节拍器 OFF / 节奏型 OFF / 和弦 ON → 回退到 V2.0 匀速琶音模式

### 3.4 方向感知扫弦

- **下行（↓）**：和弦构成音按音高从低到高依次触发，每音延迟 0.04s，模拟从低音弦往高音弦扫
- **上行（↑）**：和弦构成音按音高从高到低依次触发，每音延迟 0.04s，模拟从高音弦往低音弦扫
- 同一格子内的所有音在格子的时间戳开始触发
- 无扫弦标记的格子：不触发新音，让前一个音自然衰减

### 3.5 和弦切换

- **2.x 仅支持按小节切换**：每个和弦持续 N 个小节（默认 N=1），在下一小节的第 1 拍切换
- 当前和弦播放完 N 个小节后，自动进入下一个和弦
- 进行中最后一个和弦播完后，循环回到第一个和弦（与 V2.0 循环行为一致）
- 按拍切换和弦作为后续版本功能

### 3.6 节奏面板 UI

- **位置**：播放控制栏的展开区域
- **默认状态**：收起，仅显示一个展开按钮（如「节奏型」图标或文字入口）
- **展开后**，从上到下布局：
  1. **三开关行**：三个胶囊按钮（节拍器 / 节奏型 / 和弦），选中态高亮
  2. **BPM 行**：左侧滑块，右侧数字（如 `120`）
  3. **拍号行**：分段按钮（4/4 | 3/4 | 6/8）
  4. **节奏型选择**：下拉列表（显示名称，如「民谣基础扫弦」）→ 选中后在下方展示预览网格
  5. **预览网格**：
     - 多行（每拍一行或一行连续），格子内显示 ↓ / ↑ / ·
     - 拍与拍之间用竖线分隔，上方标拍号数字（1 / 2 / 3 / 4）
     - 播放时当前格子高亮
     - 强拍（第 1 拍）底色微区分
- **节奏型 ON 时**：V2.0 的和弦时长滑块隐藏，被 BPM 滑块和拍号选择器取代
- **节奏型 OFF 时**：和弦时长滑块恢复显示，BPM/拍号/节奏型预览隐藏

### 3.7 与 V2.0 播放的切换

- 节奏型开关 OFF：保持 V2.0 `playProgression()` 行为 —— 每个和弦按 `chordDuration` 秒匀速琶音，setInterval 循环
- 节奏型开关 ON：改用新的播放调度 —— 以 BPM 为时钟源，按节拍网格触发扫弦事件，`chordDuration` 由 `60/BPM × 每小节拍数 × 小节数` 隐式计算
- 切换发生在下次播放启动时，不打断正在进行的播放

### 3.8 导出/导入

#### 3.8.1 导出流程

- **入口**：收藏列表（创作工作台）工具栏的「导出」按钮
- **管理模式**：点击导出按钮后，收藏列表进入管理模式，每个进行左侧显示复选框，顶部显示「全选/取消」
- **选择与打包**：勾选后点击「导出选中」按钮 → 自动生成默认文件名 `chord-inspiration-YYYY-MM-DD.zip`（用户可修改）→ 打包 zip → 触发浏览器下载
- **打包内容**：每个选中进行一个文件夹，文件夹名格式为 `1-5-6-4 进行` 或 `1-4-3-6-2-5-1 进行（卡农进行）`
- **每个文件夹内**：
  - `chords.json` — 机器可读的完整数据进行备份，供导入恢复
  - `chords.md` — 人类可读的 Markdown 格式，包含进行名称、调性、和弦表、示例歌曲
  - `recordings/` — 录音文件，命名沿用录音 ID

#### 3.8.2 导入流程

- **入口**：收藏列表工具栏的「导入」按钮
- **文件选择**：弹出 `<input type="file" accept=".zip">`，仅接受 zip 文件
- **预览**：选择 zip 后解析，弹出预览弹窗，列出 zip 内所有进行
  - 每项标注该进行是否已在本地收藏中存在（冲突检测依据：progression ID 匹配）
  - 有冲突的进行提供三个操作选项：**覆盖**（用导入的笔记/录音替换本地）、**跳过**（保留本地，不导入该项）、**保留两份**（导入的数据作为新项加入收藏）
- **执行导入**：用户确认后，逐项写入 localStorage（收藏列表 + 笔记）和 IndexedDB（录音 Blob）
- **完成提示**：显示摘要（如「成功导入 3 个进行，跳过 1 个，保留两份 0 个」）

### 3.9 钢弦吉他音色

- **切换方式**：播放栏「循环播放」按钮和「记谱」切换之间新增双态点击按钮，显示 `电子 / 吉他`，点击即切换，选中态高亮
- **音色实现**：使用 Web Audio 的 **Karplus-Strong 拨弦合成算法**——噪声脉冲触发 → 延迟线 + 低通滤波反馈，模拟钢弦的物理振动过程
- **电子音色**：沿用 V2.0 的三角波 + 低通逻辑，不做改动
- **切换效果**：切换后即时生效，不影响正在播放的声音，下次播放时应用新音色
- **兼容**：节奏型 ON/OFF 两种播放模式均受音色切换影响

### 3.10 和弦发声验证

- **背景**：当前 V2.0 的和弦发声（三角波 + 低通合成）是直接基于 MIDI 音高计算生成的，尚未经过系统性的音准验证
- **验证范围**：
  - 所有和弦品质（13 种）的和弦构成音半音偏移是否正确
  - 12 个调之间的移调逻辑是否产生正确的音符拼写和 MIDI 音高
  - 音域折叠逻辑（`while (note < 36) note += 12; while (note > 80) note -= 12;`）是否导致某些和弦听起来倒置
  - 即将实现的 Karplus-Strong 钢弦吉他音色是否在正确的音高上发声
- **验证方式**：对照乐理公式和参考音频，逐项确认每个和弦发声与理论音高一致
- **优先级**：P0 — 在所有音频相关的开发工作（节奏型、吉他音色）之前完成，确保发声基座正确

### 5.2 节奏型数据结构

```typescript
type TimeSignature = '4/4' | '3/4' | '6/8';
type Subdivision = 4 | 3;  // 4 = 16th notes, 3 = triplet eighths
type StrumSlot = 'down' | 'up' | null;

interface BeatGrid {
  subdivision: Subdivision;
  slots: StrumSlot[];
}

interface RhythmPattern {
  id: string;
  name: string;
  timeSignature: TimeSignature;
  beats: BeatGrid[];
}

interface RhythmState {
  metronomeOn: boolean;
  rhythmOn: boolean;
  chordsOn: boolean;
  bpm: number;                    // 40-200, default 100
  timeSignature: TimeSignature;   // default '4/4'
  currentPattern: string | null;  // pattern id
  barRepeat: number;              // bars per chord, default 1
  isPlaying: boolean;
  countInRemaining: number;       // remaining beats in count-in, -1 = not counting in
  currentBeat: number;            // 0-based beat index within current bar
  currentSlot: number;            // 0-based slot index within current beat
}
```

### 5.3 播放调度引擎

- **时钟源**：使用 `setInterval` 以 `(60 / BPM / 每拍子分数)` 为间隔，每个 tick 推进一个格子
  - 4/4 拍 × 十六分音符 = 每拍 4 格子，每小节 16 格子
  - 3/4 拍 × 十六分音符 = 每拍 4 格子，每小节 12 格子
  - 6/8 拍 × 三连音 = 每拍 3 格子，每小节 18 格子
- **格子事件**：每个 tick 触发：
  1. 如果是拍子第一格（slot=0）→ 播放节拍器 click（强拍或弱拍）
  2. 如果节奏型 ON 且当前格子有 strum 标记 → 触发方向扫弦（和弦 ON 则带和弦音，否则轻触发）
  3. 如果是最后一小节的最后一格 → 切换下一个和弦（或循环）
  4. 推进 `currentBeat` 和 `currentSlot`
- **预备拍**：开始播放时 `countInRemaining = 每小节拍数`，在此期间仅触发节拍器 click，不计和弦/节奏型。倒计时结束后自动进入正常播放。

### 5.4 扫弦音频改动

- 在现有 `playChord()` 基础上增加方向参数
- **下行**：MIDI notes 按升序排列（低→高），每音延迟 `index * 0.04s`
- **上行**：MIDI notes 按降序排列（高→低），每音延迟 `index * 0.04s`
- 当格子有 strum 标记但和弦 OFF 时：播放一个极短的 muted strum 声（低音量噪声），让用户感知节奏方向
- 当节奏型 OFF 时：完全保持 V2.0 的 `playChord()` 行为，不做任何改动

### 5.5 节拍器音色合成

- **强拍 click**：正弦波 1200Hz，Attack 1ms → Decay 40ms → 静音，峰值音量 0.3
- **弱拍 click**：正弦波 800Hz，Attack 1ms → Decay 30ms → 静音，峰值音量 0.2
- 纯节拍器模式（和弦 OFF）：峰值音量 × 1.3
- 混合模式（和弦 ON）：峰值音量 × 0.8，避免盖过和弦

### 5.6 面板折叠/展开

- 播放栏行内新增入口图标（如 `Metronome` 或自定义图标）
- 点击展开 → Framer Motion spring 动画向下展开节奏面板
- 展开后入口图标变为选中态
- 再次点击或点击面板外区域收起
- 展开状态不持久化，每次刷新默认收起

### 5.7 预览网格渲染

- 每个格子为一个 `span` 或 `div`，宽高约 24-28px
- 内容：`↓` / `↑` / `·`（空拍用中点表示）
- 拍间竖线：`border-left` 或独立分隔符元素
- 拍号数字标注在每个拍子的第一个格子上方
- 当前播放格子：背景色高亮（使用绿色强调色 `#4f6d3a`），transition 平滑过渡
- 强拍（第 1 拍）底色：微弱的背景色区分（如 `bg-stone-100`）

### 5.8 导出/导入模块

| 模块 | 职责 | 新增/修改 | 依赖 |
|------|------|-----------|------|
| `FavExport` | 管理模式 UI（进入选择态、复选框、全选/取消）、导出/导入按钮入口 | 新增 | React state |
| `ZipPacker` | 接受选中进行列表 → 从 localStorage 拉笔记 → 从 IndexedDB 拉录音 Blob → 用 jsZip 组装 zip → 触发下载 | 新增 | jsZip, IndexedDB |
| `ZipParser` | 接受用户选择的 zip → 解析 → 提取进行列表 + 冲突检测 → 返回预览数据 | 新增 | jsZip |
| `ImportEngine` | 接受用户的导入确认 → 写回 localStorage（收藏+笔记）→ 写回 IndexedDB（录音 Blob）→ 返回完成摘要 | 新增 | IndexedDB |

模块依赖关系：

```
用户操作
  ├──「导出」→ FavExport(管理模式) → ZipPacker(打包) → 下载
```

**数据流：**
- **导出**：`localStorage.getItem('and_xian_favorite_progressions')` → 匹配选中项 → `localStorage.getItem('and_xian_progression_notes')` 获取笔记 → `getAudioMemo(id)` 从 IndexedDB 逐条拉取录音 Blob → jsZip 组装 → `URL.createObjectURL` + `<a download>` 触发下载
- **导入**：`<input type="file">` 获取 File → jsZip 解析 → 预览确认 → `setFavorites()` 更新收藏 → 写回 localStorage 笔记 → `saveAudioMemo(id, blob)` 逐条写回 IndexedDB

### 5.9 音色切换

| 模块 | 职责 | 新增/修改 | 依赖 |
|------|------|-----------|------|
| `audio.ts` — `AudioSynth.setTone()` | 接受 `'synth'` / `'guitar'` 参数，切换 `playChord()` 内部的合成分支 | 修改 | AudioContext |
| `App.tsx` — `toneMode state` | 管理当前音色状态，传递给播放栏 UI 和 synth 实例 | 修改 | React state |

**Karplus-Strong 实现概要：** 每个吉他和弦音分解为独立拨弦事件，使用 Web Audio 的延迟线（`DelayNode` + `GainNode` 反馈回路）配合低通滤波模拟钢弦振动。每个音高由延迟时间决定（`sampleRate / frequency`），反馈系数控制衰减速度。

**音色切换不影响播放状态：** 切换只改 `synth.toneMode` 标志位，不中断正在播放的声音。下次 `playChord()` 调用时按新模式合成。

## 7. 假设与风险

### 7.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|------|---------|-----------|
| 节拍器+节奏型能显著提升试听体验和回访率 | 观察节奏型使用率（目标≥30%） | 简化交互，降低入门门槛 |
| 15-20 套内置 rhythm pattern 能满足初期需求 | 观察节奏型切换率（目标≥20%） | 后续版本扩展内置库或加入自定义编辑 |
| 用户能理解三开关自由组合的逻辑 | 观察开关使用模式和用户反馈 | 加入预设模式快捷切换（纯节拍器/练扫弦/完整试听） |
| 用户有跨设备迁移收藏的需求 | 观察导出/导入使用率 | 降低导出操作门槛，若使用率低则仅做手动备份功能 |

### 7.2 已知风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| `setInterval` 时钟精度不够（JS 定时器漂移） | 中 | 中 | 使用 `performance.now()` 或 Web Audio `currentTime` 校准，积累误差超过阈值时自动修正 |
| 节拍器 + 和弦同时播放的复音数控制 | 低 | 低 | 延续 V2.0 的 gain 均分机制，节拍器 gain 独立控制 |
| 老用户不习惯节奏面板折叠交互 | 低 | 低 | 首次进入加入引导提示（如 pulsating dot 标注展开入口） |
| 录音文件较大导致 zip 包体积过大 | 中 | 低 | 不做体积限制，打包时在 UI 上给出大概预计大小 |
| IndexedDB 空间不足导致导入失败 | 低 | 中 | 导入前检查 IndexedDB 剩余配额，不足时提示用户清理 |
