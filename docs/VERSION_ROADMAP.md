# 版本路线

> 项目版本规划，跨版本参考文档。

## 版本号规则

采用三段式语义化版本号 `X.Y.Z`，详见 `~/.claude/rules/versioning.md`。

**重大节点**：X.0 版本（如 1.0、2.0、3.0），代表产品形态或技术基座的质变。重大节点之间不做详细的 Y 版本排期，待进入该 X 版本后再规划。

---

| 版本 | 状态 | 一句话 |

## 2.x 功能池

2.x 系列已按计划拆分为独立版本：

| 版本 | 状态 | 范围 | 文档 |
|------|------|------|------|
| **V2.1** | 开发中 | 节奏型系统：节拍器 + 15 套扫弦 pattern + 三模式协同 + 方向感知扫弦 + 节奏面板 + 预备拍 | [V2.1-PRD](./V2.1-PRD.md) |
| **V2.2** | 规划中 | 导出/导入（jsZip .zip 打包）+ 钢弦吉他音色（Karplus-Strong 合成） | [V2.2-PRD](./V2.2-PRD.md) |

### V2.1 — 节奏型系统

- **节拍器底座**：BPM 40–200 可调，支持 4/4、3/4、6/8 拍号，纯节拍器模式下独立使用
- **扫弦 pattern**：底层时值网格存储节奏，UI 展示方向箭头（↓↑）
- **内置库**：15 套经典节奏型（11 套 4/4 + 2 套 3/4 + 2 套 6/8）
- **三模式协同**：
  - 模式 A：纯节拍器 — 自己拿琴跟着练
  - 模式 B：节拍器 + 节奏型 — 练扫弦手感
  - 模式 C：节拍器 + 节奏型 + 和弦 — 完整试听
- 节奏型启动时取代现有匀速琶音模式
- **远期**：自定义节奏型编辑器

### V2.2 — 导出/导入 + 音色

- **导出**：收藏列表多选 → 打包 .zip
- zip 结构：每个进行一个文件夹，内含 `chords.json` + `chords.md` + `recordings/`
- **导入**：选 .zip → 预览 → 冲突处理（覆盖/跳过/保留两份）→ 恢复收藏、笔记、录音到浏览器存储
- 使用 jsZip 库，纯前端实现
- **音色**：播放栏电子 ↔ 钢弦吉他（Karplus-Strong）双态切换

### 候选功能（待讨论确认）

- 自定义节奏型编辑器（可能放到较晚的 2.Y）
- 更多和弦进行数据扩展
- iPad/桌面端响应式适配
- 更多乐器音色选项

## 演进逻辑

```
Web 功能迭代 → PWA 安装体验 → 数据同步后端 → 跨平台 App
  (V2.1/V2.2)       (V3.0)          (V4.0)         (V5.0)
```

核心原则：**重大节点升得慢**，大部分功能演进在同一个 X 版本内完成。只有产品形态发生质变时才跳下一个重大节点。
