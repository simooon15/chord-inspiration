## 变更背景

V2.0 做了一个完整创作工作台：能听和弦、看指法、写笔记、录哼唱、管理收藏。用户打开网页就能完成从「获得灵感」到「记录灵感」的闭环。但上线后，试听体验有两个明显不足：

**1. 试听是「平的」。** V2.0 的匀速琶音能展示和弦色彩，但真实音乐不是每个音等时长地往上弹。用户听到一组进行后，脑子里浮现的是一个有律动的伴奏，而 V2.0 给出的是一串机械的琶音——信息量不够。用户需要的不是「这四个音是什么」，而是「用这个节奏弹这四个和弦是什么感觉」。

**2. 练琴场景缺位。** V2.0 的设计场景是「浏览灵感」，但大量用户的使用场景是「拿着琴跟着弹」。他们需要一个节拍器来保持速度，需要扫弦节奏型来练习手感——但这些在 V2.0 里都没有。用户只能一边开着和弦灵感，一边另外打开节拍器 App，体验割裂。

**决策：优先做节奏型系统。** 节奏型系统不是「加一个功能」，而是改变了试听的产品形态——从匀速琶音升级为有律动的完整伴奏。节拍器底座让它在练琴场景下独立可用。在 V2.0 的工程基座上增量构建，不改变技术架构。

导出/导入和钢弦吉他音色放入 [V2.2-PRD](./V2.2-PRD.md)。

## 1. 产品概述

### 1.1 一句话定位

给创作者加上可调节的节拍器和扫弦节奏型，让和弦试听从「匀速琶音」升级为「有律动的完整伴奏」。

### 1.2 谁需要它

与 V2.0 一致，目标用户未变。核心场景：

- 用户在 V2.0 里听到一组和弦进行，觉得不错，但不知道实际弹出来是什么节奏感。匀速琶音只能听和声色彩，听不出律动。
- 用户拿起吉他想跟着练，需要一个节拍器。V2.0 没有节拍器，用户只能另外开一个节拍器 App。
- 用户想练特定风格的扫弦（民谣 ↓↓↑、摇滚强拍等），V2.0 无法提供节奏型参考。

### 1.3 核心价值

三个独立开关自由组合：

- **节拍器**：木鱼音色，强拍区分，BPM 40-200 可调，支持 4/4、3/4、6/8 拍号
- **节拍器 + 节奏型**：在节拍器基础上叠加扫弦方向（↓↑），练扫弦手感
- **节拍器 + 节奏型 + 和弦**：完整试听——在某个节奏型下听到和弦进行，模拟真实弹奏的律动

节奏型 ON 时，取代 V2.0 的匀速琶音模式。

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 节奏型使用率 | ≥30% | 试听时开启节奏型的用户占比 |
| 节拍器独立使用率 | ≥15% | 仅开启节拍器（不叠加和弦）的用户占比 |
| BPM 调节率 | ≥40% | 开启后调整过 BPM 的用户占比 |
| 节奏型切换率 | ≥20% | 开启后切换过不同节奏型的用户占比 |

### 1.5 不做什么

- 不做自定义节奏型编辑器（远期）
- 不做多小节节奏型
- 不做节奏型按风格分类
- 不做按拍切换和弦（v1 仅按小节切换）
- 不做导出/导入（放入 V2.2）
- 不做钢弦吉他音色（放入 V2.2）

| 和弦发声验证 | 验证 13 种品质 + 12 调移调 + 音域折叠逻辑 | 采样类验证工具 |

## 3. 功能详情

### 3.1 和弦发声验证（P0 前置）

在所有音频功能开发之前，系统性验证：

- 13 种和弦品质的半音偏移公式是否正确
- 12 个调之间的移调逻辑是否产生正确的音符拼写和 MIDI 音高
- 音域折叠逻辑（`while (note < 36) note += 12; while (note > 80) note -= 12;`）是否导致某些和弦听起来倒置
- 对照乐理公式和参考音频，逐项确认每个和弦发声与理论音高一致

### 3.2 节拍器底座（P0）

- 通过 Web Audio API 合成的短促木鱼/鼓边音色
  - 强拍（第 1 拍）：频率稍高（约 1200Hz），响度稍大
  - 弱拍（其余拍）：频率稍低（约 800Hz），响度稍小
  - 使用正弦波 + 极短 Attack/Decay 包络模拟打击乐瞬态
- **BPM 控制**：水平滑块 + 右侧数字显示，范围 40-200，步长 1
  - 拖动滑块粗调，数字实时跟随
  - 默认 BPM：100
- **拍号选择器**：分段按钮，默认 4/4，可选 3/4、6/8
  - 切换拍号后：节拍器强拍间隔、预览网格列数、节奏型列表过滤（仅显示匹配拍号）联动变化
- **预备拍**：点击播放后先播放 1 小节纯节拍器（不叠加和弦和节奏型），之后自动进入选定模式

### 3.3 节奏型数据模型

- **拍级数组**：一个小节 = `beats[]`，每个 beat 独立定义 `subdivision`（4 = 十六分音符，3 = 三连音）和 `slots[]`（每个格子为 `'down' | 'up' | null`）
- **拍号内嵌**：每个节奏型数据自带 `timeSignature` 字段，选中时自动同步拍号选择器
- **内置库**：15 套节奏型，覆盖常见的 4/4、3/4、6/8 扫弦 pattern

| 拍号 | 数量 | 典型 pattern |
|------|------|-------------|
| 4/4 | 11 | 民谣基础、民谣轻柔、摇滚强拍、流行标准、流行八分交替、流行抒情、雷鬼反拍、Funk 切分、Disco 四拍、民谣蓝草、摇滚 shuffle |
| 3/4 | 2 | 三拍子华尔兹、三拍子轻柔 |
| 6/8 | 2 | 6/8 舒缓、6/8 流动 |

- **数据只读**：V2.1 不提供编辑功能，用户仅可浏览和选择

示例数据结构：

```
{
  id: "folk_basic",
  name: "民谣基础扫弦",
  timeSignature: "4/4",
  beats: [
    { subdivision: 4, slots: [down, null, null, null] },
    { subdivision: 4, slots: [down, null, up,   null] },
    { subdivision: 4, slots: [down, null, null, null] },
    { subdivision: 4, slots: [down, null, up,   null] },
  ]
}
```

### 3.4 三开关模式

- 三个独立开关按钮，位于节奏面板顶部
  - **节拍器**：ON → 播放 click 声 + 预备拍；OFF → 静音
  - **节奏型**：ON → 按选定 pattern 的节奏触发扫弦，取代匀速琶音；OFF → 回退到 V2.0 匀速琶音（仅当和弦开关 ON 时有效）
  - **和弦**：ON → 播放和弦进行音色；OFF → 仅节拍器 + 节奏型方向（无声扫弦或不触发音高）
- 开关状态互不锁定，自由组合
- 典型组合：
  - 节拍器 ON / 节奏型 OFF / 和弦 OFF → 纯节拍器
  - 节拍器 ON / 节奏型 ON / 和弦 OFF → 节拍器 + 可视节奏型（练扫弦）
  - 节拍器 ON / 节奏型 ON / 和弦 ON → 完整试听
  - 节拍器 OFF / 节奏型 OFF / 和弦 ON → 回退到 V2.0 匀速琶音模式

### 3.5 方向感知扫弦

- **下行（↓）**：和弦构成音按音高从低到高依次触发，每音延迟 0.04s，模拟从低音弦往高音弦扫
- **上行（↑）**：和弦构成音按音高从高到低依次触发，每音延迟 0.04s，模拟从高音弦往低音弦扫
- 同一格子内的所有音在格子的时间戳开始触发
- 无扫弦标记的格子：不触发新音，让前一个音自然衰减

### 3.6 和弦切换

- **V2.1 仅支持按小节切换**：每个和弦持续 N 个小节（默认 N=1），在下一小节的第 1 拍切换
- 当前和弦播放完 N 个小节后，自动进入下一个和弦
- 进行中最后一个和弦播完后，循环回到第一个和弦（与 V2.0 循环行为一致）
- 按拍切换和弦作为后续版本功能

### 3.7 节奏面板 UI

- **位置**：播放控制栏的展开区域
- **默认状态**：收起，仅显示一个展开按钮（节拍器图标）
- **展开后**，从上到下布局：
  1. **三开关行**：三个胶囊按钮（节拍器 / 节奏型 / 和弦），选中态高亮
  2. **BPM 行**：左侧滑块，右侧数字（如 `120`）
  3. **拍号行**：分段按钮（4/4 | 3/4 | 6/8）
  4. **节奏型选择**：下拉列表（显示名称，如「民谣基础扫弦」）→ 选中后在下方展示预览网格
  5. **预览网格**：
     - 一行连续格子，每个格子宽高约 24-28px
     - 格子内显示 ↓ / ↑ / ·（空拍用中点表示）
     - 拍与拍之间用竖线分隔，上方标拍号数字（1 / 2 / 3 / 4）
     - 播放时当前格子高亮（绿色 `#4f6d3a`）
     - 强拍（第 1 拍）底色微区分
- **节奏型 ON 时**：V2.0 的和弦时长滑块隐藏，被 BPM 滑块和拍号选择器取代
- **节奏型 OFF 时**：和弦时长滑块恢复显示，BPM/拍号/节奏型预览隐藏

### 3.8 与 V2.0 播放的切换

- 节奏型开关 OFF：保持 V2.0 `playProgression()` 行为 —— 每个和弦按 `chordDuration` 秒匀速琶音，setInterval 循环
- 节奏型开关 ON：改用新的播放调度 —— 以 BPM 为时钟源，按节拍网格触发扫弦事件，`chordDuration` 由 `60/BPM × 每小节拍数 × 小节数` 隐式计算
- 切换发生在下次播放启动时，不打断正在进行的播放

## 4. 验收标准

### US-1 (P0)：纯节拍器使用

**As a** 拿起吉他想跟着节拍练琴的用户
**I want to** 打开节拍器，调节 BPM 和拍号
**so that** 不用另外开节拍器 App，在同一个页面里完成

- **Scenario 打开节拍器**
  - Given 在页面中展开节奏面板
  - When 开启「节拍器」开关（节奏型 OFF，和弦 OFF）
  - Then 开始播放木鱼 click 声，强拍（第 1 拍）音高/响度区分于弱拍

- **Scenario 调节 BPM**
  - Given 节拍器正在播放
  - When 拖动 BPM 滑块从 100 到 120
  - Then 数字显示实时变为 120，click 间隔缩短对应比例

- **Scenario 切换拍号**
  - Given 当前拍号为 4/4
  - When 切换到 3/4
  - Then 强拍间隔从 4 拍变为 3 拍，click 声相应调整

### US-2 (P0)：节奏型试听

**As a** 想练特定扫弦节奏的吉他爱好者
**I want to** 选一套扫弦节奏型并听到方向感清晰的扫弦
**so that** 理解这个节奏型的手感和听感

- **Scenario 选择节奏型**
  - Given 节奏面板已展开
  - When 开启「节拍器」+「节奏型」开关，从下拉列表选「民谣基础扫弦」
  - Then 预览网格显示对应箭头 pattern，播放时按 ↓ 低→高、↑ 高→低的顺序扫弦

- **Scenario 切换节奏型**
  - Given 正在播放「民谣基础扫弦」
  - When 从下拉列表切换到「摇滚强拍」
  - Then 预览网格更新为新 pattern，下次小节开始时生效

- **Scenario 拍号联动**
  - Given 当前拍号为 4/4
  - When 切换到 3/4 拍号
  - Then 节奏型下拉列表过滤为仅 3/4 的 pattern

### US-3 (P0)：完整试听（节拍器+节奏型+和弦）

**As a** 听到一组和弦进行但想知道实际弹奏律动的用户
**I want to** 在节奏型下听到和弦进行
**so that** 感受到有节奏感的完整伴奏

- **Scenario 完整试听**
  - Given 已生成一组和弦进行
  - When 三开关全开（节拍器 + 节奏型 + 和弦），点击播放
  - Then 先播放 1 小节预备拍，之后按选定节奏型依次扫出和弦，每个和弦持续 1 小节

- **Scenario 节奏型 OFF 回退**
  - Given 正在以节奏型模式播放
  - When 关闭「节奏型」开关
  - Then 下次播放启动时回到 V2.0 匀速琶音模式，和弦时长滑块恢复显示

### US-4 (P1)：和弦按小节切换

**As a** 试听和弦进行的用户
**I want to** 每个和弦在完整小节边界上切换
**so that** 切换时机自然对齐节拍，不突兀

- **Scenario 小节切换**
  - Given 正在播放 4/4 节奏型，和弦进行有 4 个和弦
  - When 播放进行到第 1 小节的最后 1 拍
  - Then 下一小节的第 1 拍切换到下一个和弦

- **Scenario 循环播放**
  - Given 最后一个和弦已播完
  - When 到达该和弦最后一个小节末尾
  - Then 自动循环到第一个和弦，从第 1 拍开始

### US-5 (P1)：预览网格高亮

**As a** 跟着节奏型练琴的用户
**I want to** 在预览网格上看到当前播放位置
**so that** 知道现在弹到第几拍第几个子分

- **Scenario 播放高亮**
  - Given 节奏型正在播放
  - When 当前处于第 2 拍第 3 个十六分格子
  - Then 预览网格中对应格子高亮（绿色背景 `#4f6d3a`）

### US-6 (P2)：预备拍

**As a** 想跟着节拍进和弦的用户
**I want to** 在正式播放前有 1 小节预备拍
**so that** 能找准速度，在正确的时机进入

- **Scenario 预备拍**
  - Given 三开关全开
  - When 点击播放按钮
  - Then 先播放 1 小节纯节拍器（无和弦无节奏型），之后自动进入完整播放

---

### US-7 (P0)：和弦发声验证

**As a** 依赖和弦正确发声的功能开发者
**I want to** 系统性验证所有和弦品质和调性下的发声正确性
**so that** 确保音高基座准确，节奏型功能建立在正确之上

- **Scenario 验证和弦品质**
  - Given 13 种和弦品质各有对应的半音偏移公式
  - When 对每个品质选取多个根音进行发声
  - Then 发出的 MIDI 音高与乐理理论音高一致

- **Scenario 验证移调正确性**
  - Given 一段和弦进行在 C 调下发声正确
  - When 依次切换到全部 12 个调
  - Then 每个调下的和弦名拼写和发声音高正确

- **Scenario 验证音域折叠**
  - Given MIDI 音符超出 36-80 范围时会被折叠八度
  - When 和弦包含极高或极低的构成音
  - Then 折叠后的音高仍在合理范围内，且和弦倒置感不明显

## 5. 技术设计

### 5.1 模块拆分

| 模块 | 职责 | 新增/修改 | 依赖 |
|------|------|-----------|------|
| `RhythmEngine` | BPM 时钟调度、拍子计数、预备拍、小节循环、播放状态机 | 新增 | Web Audio 时钟 |
| `PatternData` | 节奏型数据结构定义 + 15 套内置 pattern | 新增 | 无 |

### 5.2 节奏型数据结构

```typescript
type TimeSignature = '4/4' | '3/4' | '6/8';
type Subdivision = 4;  // 4 = 16th notes
type StrumSlot = 'down' | 'up' | null;

interface BeatGrid {
  subdivision: Subdivision;
  slots: StrumSlot[];
}

interface RhythmPattern {
  id: string;
  name: string;
  timeSignature: TimeSignature;
  beats: BeatGrid[];
}
```

### 5.3 播放调度引擎

- **时钟源**：使用 `setInterval` 以 `(60 / BPM / 4)` 为间隔（十六分音符），每个 tick 推进一个格子
  - 4/4 拍 × 十六分音符 = 每拍 4 格子，每小节 16 格子
  - 3/4 拍 × 十六分音符 = 每拍 4 格子，每小节 12 格子
  - 6/8 拍 × 十六分音符 = 每拍 4 格子，每小节 24 格子
- **格子事件**：每个 tick 触发：
  1. 如果是拍子第一格（slot=0）→ 播放节拍器 click（强拍或弱拍）
  2. 如果节奏型 ON 且当前格子有 strum 标记 → 触发方向扫弦（和弦 ON 则带和弦音，否则轻触发）
  3. 如果是最后一小节的最后一格 → 切换下一个和弦（或循环）
  4. 推进 `currentBeat` 和 `currentSlot`
- **预备拍**：开始播放时先进行 1 小节纯节拍器，在此期间仅触发节拍器 click。倒计时结束后自动进入正常播放。

### 5.4 扫弦音频改动

- 在现有 `playChord()` 基础上增加方向参数
- **下行**：MIDI notes 按升序排列（低→高），每音延迟 `index * 0.04s`
- **上行**：MIDI notes 按降序排列（高→低），每音延迟 `index * 0.04s`
- 当格子有 strum 标记但和弦 OFF 时：播放一个极短的 muted strum 声（低音量噪声），让用户感知节奏方向
- 当节奏型 OFF 时：完全保持 V2.0 的 `playChord()` 行为，不做任何改动

### 5.5 节拍器音色合成

- **强拍 click**：正弦波 1200Hz，Attack 1ms → Decay 40ms → 静音，峰值音量 0.3
- **弱拍 click**：正弦波 800Hz，Attack 1ms → Decay 30ms → 静音，峰值音量 0.2
- 纯节拍器模式（和弦 OFF）：峰值音量 × 1.3
- 混合模式（和弦 ON）：峰值音量 × 0.8，避免盖过和弦

### 5.6 面板折叠/展开

- 播放栏行内新增入口图标（节拍器图标）
- 点击展开 → Motion spring 动画向下展开节奏面板
- 展开后入口图标变为选中态
- 再次点击收起
- 展开状态不持久化，每次刷新默认收起

### 5.7 预览网格渲染

- 每个格子为一个 `span` 或 `div`，宽高约 24-28px
- 内容：`↓` / `↑` / `·`（空拍用中点表示）
- 拍间竖线：`border-left` 或独立分隔符元素
- 拍号数字标注在每个拍子的第一个格子上方
- 当前播放格子：背景色高亮（使用绿色强调色 `#4f6d3a`），transition 平滑过渡
- 强拍（第 1 拍）底色：微弱的背景色区分（如 `bg-stone-100`）

## 6. 与 V2.0 的差异总结

| 维度 | V2.0 | V2.1 |
|------|------|------|
| 试听节奏 | 匀速琶音，每和弦等时长 | 节奏型驱动，方向感知扫弦 |
| 节拍器 | 无 | Web Audio 合成木鱼音色，强拍区分 |
| BPM/拍号 | 无，仅和弦时长滑块（1.2-2.6s） | BPM 40-200 滑块 + 4/4、3/4、6/8 拍号 |
| 扫弦方向 | 仅低→高（固定） | ↓ 低→高、↑ 高→低，音频区分 |
| 和弦切换 | 固定时长 interval | 按小节边界切换（v1） |
| 声音层次 | 单层（合声音） | 双层（节拍器 + 合声音），独立开关 |
| 预备拍 | 无 | 1 小节 count-in |
| 播放面板 | 播放/停止 + 速度滑块 | 折叠式节奏面板，三开关 + BPM + 拍号 + 节奏型选择 |
| 预览网格 | 无 | 方向箭头 + 拍子分隔 + 播放高亮 |

## 7. 假设与风险

### 7.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|------|---------|-----------|
| 节拍器+节奏型能显著提升试听体验和回访率 | 观察节奏型使用率（目标≥30%） | 简化交互，降低入门门槛 |
| 15 套内置 rhythm pattern 能满足初期需求 | 观察节奏型切换率（目标≥20%） | 后续版本扩展内置库或加入自定义编辑 |
| 用户能理解三开关自由组合的逻辑 | 观察开关使用模式和用户反馈 | 加入预设模式快捷切换（纯节拍器/练扫弦/完整试听） |

### 7.2 已知风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| `setInterval` 时钟精度不够（JS 定时器漂移） | 中 | 中 | 使用 Web Audio `currentTime` 校准，积累误差超过阈值时自动修正 |
| 节拍器 + 和弦同时播放的复音数控制 | 低 | 低 | 延续 V2.0 的 gain 均分机制，节拍器 gain 独立控制 |
| 老用户不习惯节奏面板折叠交互 | 低 | 低 | 首次进入加入引导提示 |
