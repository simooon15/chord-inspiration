## 1. 产品概述

### 1.1 产品背景

音乐创作者（吉他/钢琴/尤克里里爱好者、独立唱作人）在创作时经常面临「拿起乐器不知道弹什么」的困境。现有方案——翻老歌、刷 YouTube 教程、翻和弦书——流程太长，找灵感 30 分钟、真正弹 10 分钟，严重打断创作心流。

### 1.2 产品定位

**一个「灵感启动器」而非「和弦工具」。** 单页网页应用，用户选调性、选风格、点一下按钮，立刻看到一组随机和弦进行（含级数标注）。不需要登录、不需要设置、打开就能用。

### 1.3 目标用户

| 用户画像 | 特征 | 核心痛点 |
|---|---|---|
| 卧室唱作人 | 会弹吉他/钢琴，偶尔写歌，乐理基础薄弱 | 不知道从哪开始，总在同样套路里循环 |
| 即兴爱好者 | 碎片时间摸琴，追求好玩和新鲜 | 找灵感的过程太慢，打断心流 |
| 乐理学习者 | 正在学和弦，想通过实例理解进行逻辑 | 看不懂和弦之间的相对关系 |

### 1.4 产品目标

| 指标 | 目标值 | 衡量方式 |
|---|---|---|
| 核心操作完成率 | ≥90% 用户能完成一次生成 | 生成按钮点击 / 页面打开 |
| 重复使用率 | ≥50% 用户会再点一次生成 | 同一会话内连续生成次数 |
| 操作效率 | 核心流程 ≤3 步 | 按验收条件逐条验证 |

## 2. 功能范围

### 2.1 MVP (V1.0) In / Out

| 维度 | MVP 做 | MVP 不做 |
|---|---|---|
| 调性选择 | 顶部下拉框，12 个调，默认 C 调并明确标注 | 自动检测调性 |
| 风格选择 | 5 种（流行/民谣/摇滚/爵士/R&B） | 自定义风格 |
| 和弦生成 | 选调+选风格 → 随机生成一组进行 | AI 生成、用户输入条件 |
| 和弦显示 | 和弦名称 + 构成音 + 纳什维尔数字级数 + 调性标注 | 五线谱、吉他指位图、音频播放 |
| 级数标注 | 纳什维尔数字（1/5/6m），后续版本加罗马数字切换 | 罗马数字（V2 加） |
| 刷新 | 一键换一组、换调性即时映射 | 撤销、历史记录 |
| 保存 | 不做（V2） | 收藏、导出、账号系统 |
| 设备适配 | 仅移动端竖屏 | iPad（V2 响应式）、桌面（暂不做） |
| 技术方案 | 纯前端 HTML/CSS/JS，无后端 | 数据库、API、用户系统 |

### 2.2 版本规划

| 版本 | 内容 | 优先级依据 |
|---|---|---|
| V1.0 MVP | P0 随机生成 + P1 多风格 + 调性/级数 + P2 极简约束 | 验证核心价值假设 |
| V2.0 | P3 收藏（localStorage） + 纳什维尔/罗马数字切换 + iPad 响应式适配 | 留存 + 覆盖 iPad 看谱用户 |
| V3.0 | P4 和弦指位图 | 降低弹奏门槛 |
| 不做 | P5 音频播放 | 用户有乐器，可自行试弹 |

## 3. 功能详情

### 3.1 调性选择器

**位置**：页面顶部，固定可见  
**交互**：`<select>` 下拉框，默认选中「C 调」，选项包含 12 个调（C / C# / D / Eb / E / F / F# / G / Ab / A / Bb / B）  
**联动**：切换调性后，当前显示的和弦即时映射到新调，无需重新点击生成  
**空状态**：无。页面打开时默认 C 调已选中  

### 3.2 风格选择器

**位置**：调性选择器下方  
**交互**：5 个风格按钮，单选，默认无选中  
**选项**：流行 / 民谣 / 摇滚 / 爵士 / R&B  
**状态**：
- 未选中：默认样式
- 选中：高亮态
- 点击另一个风格：切换选中，之前选中的取消高亮

### 3.3 生成按钮

**位置**：风格选择器下方  
**交互**：点击「给我灵感」按钮  
**前置条件**：已选择调性（默认 C 调视为已选）+ 已选择风格  
**边界情况**：如果未选风格，按钮置灰不可点击，显示提示「先选一个风格吧」  
**生成逻辑**：
1. 根据当前选中的风格，从对应数据集中随机取出一组和弦进行
2. 根据当前调性，将和弦进行中的每个和弦做音名映射（如 C→G：所有音名 +7 半音）
3. 使用纳什维尔数字标注级数（如 1-5-6m-4）
4. 在结果区域渲染输出

### 3.4 结果展示

**位置**：生成按钮下方  
**格式**：`[当前调性] [级数]: [和弦名称列表]`  
**示例**：`C 调 1-5-6m-4: C - G - Am - F`  
**每个和弦展示**：和弦名称 + 构成音（如 `Cmaj7: C-E-G-B`）  
**更新时机**：
- 点击「给我灵感」→ 生成新结果
- 切换调性 → 当前结果即时重映射（不换进行，只换 key）
- 切换风格 → 不自动生成，需再次点击「给我灵感」

### 3.5 一键刷新

**位置**：结果展示区域下方  
**交互**：点击「再换一组」按钮  
**行为**：保持当前调性和风格不变，随机生成一组新的和弦进行  
**条件**：仅在已经生成过至少一次后显示此按钮（首次生成前不显示）

### 3.6 整体布局（移动端竖屏）

```
┌─────────────────────────┐
│     🎵 和弦灵感          │  ← 页面标题
├─────────────────────────┤
│  调性: [C 调 ▾]         │  ← 下拉框，默认 C 调
├─────────────────────────┤
│  风格:                  │
│  [流行] [民谣] [摇滚]    │  ← 风格按钮组
│  [爵士] [R&B]           │
├─────────────────────────┤
│  [ ✨ 给我灵感 ]         │  ← 主操作按钮
├─────────────────────────┤
│  C 调 1-5-6m-4:         │
│  C - G - Am - F         │  ← 结果显示区
│                         │
│  C: C-E-G               │
│  G: G-B-D               │  ← 构成音展开
│  Am: A-C-E              │
│  F: F-A-C               │
│                         │
│  [ 🔄 再换一组 ]         │  ← 刷新按钮（生成后出现）
└─────────────────────────┘
```

## 4. 数据源设计

### 4.1 和弦进行数据

每种风格预置 ≥15 组和弦进行（以 C 调为基准存储），确保随机生成时有足够变化空间。数据结构：

```
{
  style: "流行",
  progressions: [
    {
      degrees: [1, 5, 6, 4],        // 纳什维尔级数
      chords: ["C", "G", "Am", "F"]  // C 调基准和弦
    },
    { degrees: [1, 6, 4, 5], chords: ["C", "Am", "F", "G"] },
    // ... 至少 15 组
  ]
}
```

### 4.2 音名映射表

12 个调的音名映射，用于调性切换时的和弦转换。例如 C→G 转换：C→G, Dm→Am, G→D, F→C，即所有和弦 +7 个半音。

## 6. 关键假设与风险

### 6.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|---|---|---|
| 用户觉得和弦数据有用 | 给 3 个玩乐器的朋友试用，问「你会再打开吗」 | 补数据，全部人工筛选 |
| 用户不需要播放就能判断和弦 | 观察朋友试用时是否有人说「能放一下吗」 | V2 加 Web Audio 预览 |
| 随机性带来惊喜而非焦虑 | 观察用户是否连续点击 ≥5 次 | 加调性过滤选项 |

### 6.2 风险

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 和弦数据太水 | 中 | 高——核心价值崩溃 | 全部人工筛选，不用 AI 生成 |
| 随机重复率高 | 中 | 中——不新鲜 | 每种风格 ≥15 组 + 简单去重 |
| 移动端浏览器兼容 | 低 | 中——打不开 | 只用标准 HTML/CSS |

## 7. 验收标准

详见 `02-user-stories.md`，核心场景摘要：

- 首次使用，默认 C 调 → 选风格 → 点生成 → 看到和弦（含级数标注），操作 ≤3 步
- 切换调性，同一组和弦即时映射到新调，无需重新生成
- 同风格连续生成 5 次，每次结果不完全相同
- 切换风格（流行→爵士），和弦特征有明显差异
- 未选风格时，「给我灵感」按钮不可点击并有提示
