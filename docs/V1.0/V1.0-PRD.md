## 1. 产品概述

### 1.1 一句话定位

给「略懂乐理但卡在套路的爱好者」一个零摩擦的创作起点。

### 1.2 谁需要它

专业的、完全不懂的，都不需要。

- **专业创作者** → 常见和弦走向烂熟于心，不需要工具提醒
- **完全不懂乐理的小白** → 连「和弦走向」是什么都不知道，没有这个需求
- **中间层——我们的用户** → 学过一点乐理，知道和弦走向的概念，听过 1-6-4-5、卡农进行，但对更丰富的走向没有积累。想创作但不知道从哪下手，拿起琴脑子一片空白

**真实状态：** 上了一段时间吉他/键盘课，练了几首弹唱，对「和弦级数」有基本概念。偶尔哼出旋律想配和弦，但只会书里那两套进行，弹出来的东西自己都觉得雷同。知道「我缺素材」但不知道去哪找。

### 1.3 核心价值

打开网页 → 选调性 → 点一下 → 看到一组随机和弦进行（含级数）。不只是告诉用户弹什么，还让他看懂它为什么这么走——这样每生成一次，就积累一个可用素材。

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 生成成功率 | ≥90% | 生成按钮点击 / 页面打开 |
| 重复使用率 | ≥50% | 同一会话内连续生成 ≥2 次的用户占比 |
| 操作步数 | 核心流程 ≤3 步 | 逐条验收 |

### 1.5 不做什么

不登录、不播放音频、不收藏导出、不做指位图、不教乐理、不支持桌面端。

## 2. 功能范围

### 2.1 In / Out

| 维度 | V1.0 做 | V1.0 不做 |
|------|---------|-----------|
| 调性 | 12 调选择器，默认 C 调 | 自动检测调性 |
| 风格 | 5 种：流行 / 民谣 / 摇滚 / 爵士 / R&B | 自定义风格、混合风格 |
| 生成 | 选调 + 风格 → 随机一组和弦进行 | 用户输入条件、AI 生成 |
| 展示 | 和弦名 + 构成音 + 级数（纳什维尔数字） | 五线谱、指位图、音频播放 |
| 交互 | 换调实时映射、一键生成、一键刷新 | 撤销、历史记录 |
| 技术 | 纯前端 HTML/CSS/JS，零依赖，和弦数据在 chords.js | 后端、数据库、API |
| 设备 | 移动端竖屏（375-414px） | iPad / 桌面端 |
| 账号 | 无 | 登录、注册、用户系统 |
| 保存 | 无 | 收藏、导出 |

### 2.2 优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 随机生成 + 调性 + 级数展示 | 没有生成就没产品 |
| P1 | 5 种风格 × 各 ≥15 组数据 | 数据不够用户用一次就腻 |
| P2 | 核心操作 ≤3 步 | 不是独立功能，是所有功能的约束条件 |

### 2.3 版本规划

| 版本 | 内容 |
|------|------|
| V1.0 | P0+P1+P2，移动端 375-414px |
| V1.0.1 | 响应式适配 375-440px，间距压缩，标题字体，点击修复 |
| V1.0.2 | 空状态优化：首次未生成时隐藏卡片区和刷新按钮 |
| 后续 | 收藏、iPad 适配、指位图等，待讨论 |

## 3. 功能详情

### 3.1 调性选择器

- 页面顶部 `<select>`，12 个调，默认 C
- 切换后已展示的和弦即时映射到新调，不重新生成

### 3.2 风格选择器

- 调性下方 `<select>`，5 种风格，默认流行
- 切换后下次生成时生效

### 3.3 「给我灵感」按钮

- 点击后根据当前调性和风格，随机抽取一组和弦进行
- 防连续重复：确保与上一次不完全相同
- 纯数组抽选，零耗时，无 loading 态

### 3.4 和弦卡片区

- 每张卡片三列布局：和弦名（左）｜ 构成音（中）｜ 级数（右）
- 四张卡片阶梯式入场动画
- 点击卡片有缩放反馈

### 3.5 「再换一组」按钮

- 保持当前调性和风格，重新随机生成一组

## 4. 验收标准

### US-1 (P0)：随机生成和弦进行

**As a** 会弹点吉他/键盘但不知道弹什么的爱好者
**I want to** 选个调性和风格，获得一组随机的和弦进行
**so that** 能立刻开始弹，不打断创作心流

- **Scenario 首次使用**
  - Given 打开网页，调性默认 C、风格默认流行，无需登录
  - When 点击「给我灵感」
  - Then 显示当前调性的一组 4 个和弦，每个附级数和构成音

- **Scenario 连续生成不重复**
  - Given 已生成过一次
  - When 再次点击「给我灵感」
  - Then 新一组和弦进行，与上一次不完全相同

### US-2 (P1)：风格差异化

**As a** 写了几首歌都觉得一个味的唱作人
**I want to** 在不同风格间切换并得到不同的和弦类型
**so that** 接触不熟悉的进行，写出更多样的作品

- **Scenario 风格差异**
  - Given 已生成过流行风格
  - When 切换到「爵士」并生成
  - Then 结果包含爵士特征和弦（七/九和弦），与流行明显不同

- **Scenario 同风格多样性**
  - Given 连续在「民谣」风格下生成 5 次
  - Then 每组至少 1 个和弦不同

### US-3 (P2)：三步内完成

**As a** 正在即兴弹琴的爱好者
**I want to** 最少操作就能获得一组和弦
**so that** 注意力始终留在乐器上

- **Scenario 最短路径**
  - Given 打开网页
  - When 点击「给我灵感」
  - Then 1 步完成（默认选项无需额外操作）

- **Scenario 需要切换选项**
  - Given 用户想换调性或风格
  - When 切换 → 点击生成
  - Then ≤3 步完成

- **Scenario 刷新当前**
  - Given 已生成过一次
  - When 点击「再换一组」
  - Then 1 步完成，保持当前调性和风格

### US-4：调性实时映射

**As a** 想用自己的调弹琴的用户
**I want to** 切换调性时已有的和弦自动跟随
**so that** 不需要重新生成就能试不同调

- **Scenario 切换调性**
  - Given 当前显示 C 调和弦进行
  - When 调性切换到 G
  - Then 同组级数自动映射，和弦名和构成音变为 G 调

## 5. 技术设计

### 5.1 数据模型

和弦数据存储在独立文件 `chords.js`，以 C 调为基准：

```js
{ degrees: [1,5,6,4], chords: ['C','G','Am','F'], notes: ['C-E-G','G-B-D','A-C-E','F-A-C'] }
```

12 调音名映射表 + `transposeChord()` / `transposeNote()` 转调函数，供页面实时调用。5 种风格各 15-16 组。

### 5.2 非功能需求

| 维度 | 要求 |
|------|------|
| 加载速度 | 首次 <2s，纯静态无网络请求 |
| 兼容性 | iOS Safari / Android Chrome 最近 2 个大版本 |
| 响应式 | 375-414px（更大屏幕居中 + 背景色，不做布局适配） |
| 可访问性 | 按钮 ≥44×44px，尊重 `prefers-reduced-motion` |
| 技术栈 | 零框架、零库、零 CDN，单文件 HTML + chords.js 数据文件 |

### 5.3 设计系统

| 变量 | 值 |
|------|-----|
| 主色 | `#4f6d3a`（森林绿） |
| 卡片圆角 | `20px` |
| 和弦名字号 | `32px` |
| 级数字号 | `17px` |
| 容器宽度 | `max-width: 414px`，水平居中 |

风格：iOS 26 Liquid Glass 玻璃质感。标题 sans-serif，和弦名衬线体。具体参数参见 design-spec。

## 6. 假设与风险

### 6.1 核心假设

| 假设 | 验证方式 | 失败应对 |
|------|---------|-----------|
| 和弦数据本身能激发灵感 | 找 3 个目标用户试用，问「会再打开吗」 | 全部人工筛选，不用 AI 生成的数据 |
| 用户看到和弦符号就能判断好听与否 | 观察是否有人说「能播放一下听听吗」 | 后续版本考虑 Web Audio 预览 |
| 随机性能带来惊喜感 | 观察用户是否连击 ≥5 次 | 加过滤规则降低重复率 |

### 6.2 已知风险

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 和弦数据质量不够 | 中 | 高 | 全部人工筛选，每种风格 ≥15 组 |
| 随机重复率偏高 | 中 | 中 | 每种 ≥15 组 + 防连续重复逻辑 |
| 移动端浏览器兼容 | 低 | 中 | 只用标准 HTML/CSS/JS |
