## 1. 产品概述

### 1.1 更新背景

V1.0 在 `max-width: 414px` 固定宽度下开发和测试。上线后发现四个问题：

- **横向不适配**：414px 固定容器在 iPhone 17（402px CSS 宽度）下表现异常
- **纵向过长**：固定间距和 padding 导致需要滚动，信息本可在一屏展示
- **标题字体单薄**：标题使用系统 sans-serif，与和弦卡片的衬线字体缺乏层次对比
- **和弦卡片点击卡顿**：`:active` 缩放动效在 iOS Safari 上响应滞后，因 `animation-fill-mode: both` 持续持有 `transform` 属性，与 `:active` 的缩放冲突

### 1.2 更新目标

1. 容器自适应 375–440px CSS 宽度
2. 纵向间距压缩，iPhone SE 竖屏一屏看完核心信息
3. 标题改用思源宋体，与卡片衬线字体统一品牌基调
4. 修复和弦卡片点击卡顿

### 1.3 影响范围

| 机型 | CSS 宽度 | V1.0 表现 | V1.0.1 目标 |
|------|---------|----------|-----------|
| iPhone SE | 375px | 可行，卡片窄 | 自适应 |
| iPhone 15/16 | 390px | 基本适配 | 自适应 |
| **iPhone 17** | **402px** | **不适配** | 自适应 |
| iPhone 17 Air | 420px | 待验证 | 自适应 |
| iPhone 17 Pro Max | 440px | 容器太窄 | 自适应 |

## 3. 功能详情

### 3.1 容器自适应

取消 `max-width: 414px` 固定值，改为 `width: 100%; max-width: 440px`，横向内边距用 `clamp(16px, 5vw, 24px)` 自适应。

### 3.2 纵向压缩

顶部留白 72→52px，控件→按钮 20→16px，按钮→和弦区 36→0（按钮 margin-bottom 归零），和弦区→刷新 36→0，底部留白 48→max(20px, env(safe-area-inset-bottom))。总高度减少约 50–60px。

### 3.3 标题字体

标题「和弦灵感」从系统 sans-serif 改为 `Noto Serif SC`（思源宋体），weight 575。通过 Google Fonts 子集化加载，仅包含页面所需五个汉字，不影响加载速度。

### 3.4 和弦卡片点击优化

**根因**：`.chord-item.animate-in` 使用 `animation: chordIn 800ms ease both`，`both` 填充模式让动画结束后持续持有 `transform: translateX(0)`。WebKit 中 CSS animation 对 `transform` 的优先级高于 `:active` 伪类，导致点击时 `scale(0.88)` 与动画持有的 `translateX(0)` 冲突，合成器产生卡顿。

**修复**：`animationend` 事件中移除 `animate-in` 类 + 清除 `animation` 属性，将 `transform` 交还给 `:active` 独占。同时删除未使用的 `.chord-item.tapped` CSS 规则。
