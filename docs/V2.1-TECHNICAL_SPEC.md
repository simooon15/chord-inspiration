## 2. 节奏型数据层

### 2.1 类型定义 (`src/rhythm/types.ts`)

```typescript
export type TimeSignature = '4/4' | '3/4' | '6/8';
export type StrumDirection = 'down' | 'up';
export type StrumSlot = StrumDirection | null;

export interface BeatGrid {
  subdivision: number;        // 每拍子分格数，4 = 十六分音符
  slots: StrumSlot[];         // 长度 === subdivision，每个格子的扫弦方向
  durations?: number[];       // 每个非 null slot 的持续时长（16分音符单位）
}

export interface RhythmPattern {
  id: string;                 // 唯一标识，如 "custom_d_ddu_ud_ddu"
  name: string;               // 中文名，如 "D DDU UD DDU"
  timeSignature: TimeSignature;
  beats: BeatGrid[];          // 长度 = beatsPerBar × barCount
  barCount: number;           // 1 = 单小节, 2 = 两小节
  // beats 数组长度示例：
  //   4/4, barCount=1 → beats[4]
  //   4/4, barCount=2 → beats[8]
  //   3/4, barCount=1 → beats[3]
  //   6/8, barCount=1 → beats[6]
}
```

### 2.2 内置数据 (`src/rhythm/patterns.ts`)

15 套扫弦节奏型（含 6 套两小节），不可编辑：

| 拍号 | barCount | 数量 | 典型 pattern |
|------|----------|------|-------------|
| 4/4 | 1 | 1 | D DDU UD DDU（自定义） |
| 4/4 | 2 | 6 | 光辉岁月、蓝莲花、Rock 变奏 1/2、Disco、流行 |
| 3/4 | 1 | 5 | 华尔兹下上/切分/弱起/花式、栀子花开 |
| 6/8 | 1 | 3 | 三连扫、六拍切分、六拍抒情 |

每个 `slots[]` 内元素为 `'down'` / `'up'` / `null`。

内置辅助函数 `slots(str)` 将字符串（'D'/'U'/'.'）转为 StrumSlot 数组，`beat(...strs)` 批量创建 BeatGrid。

**按拍号过滤：**
```typescript
export function getPatternsByTimeSignature(ts: string): RhythmPattern[] {
  return ALL_PATTERNS.filter(p => p.timeSignature === ts);
}
```

## 3. 播放调度引擎 (`src/rhythm/engine.ts`)

### 3.1 职责

替换 V2.0 的 `setInterval(chordDuration)` 简单循环，升级为以 BPM 为时钟源、按格子 tick 推进的调度器。

### 3.2 核心接口

```typescript
export class RhythmEngine {
  private bpm: number;
  private timeSignature: TimeSignature;
  private pattern: RhythmPattern | null;
  private metronomeOn: boolean;
  private rhythmOn: boolean;
  private chordsOn: boolean;
  private chordMode: 'pattern' | 'bar';  // 和弦切换模式
  private tickIntervalId: number | null;
  private currentBeat: number;     // 当前拍号索引 (0-based)
  private currentSlot: number;     // 当前格子索引 (0-based)
  private currentChordIndex: number;
  private tickCount: number;       // 总 tick 数（用于预备拍计数）

  // 回调
  onTick?: (info: TickInfo) => void;       // 每个 tick 触发
  onChordChange?: (index: number) => void; // 和弦切换时触发
  onCountInEnd?: () => void;               // 预备拍结束
}
```

### 3.3 Tick 时钟

使用 `requestAnimationFrame` 驱动 tick 循环，以 `performance.now()` 为时间基准：

```typescript
tickInterval = (60000 / (BPM × bpmFactor)) / subdivision
```

- 4/4, subdivision=4: interval = 60000 / BPM / 4 ms
- 6/8: bpmFactor=3（因为 6/8 的 BPM 以附点四分音符为单位）
- BPM=100 时，interval = 150ms（每个十六分音符 150ms）

**时钟实现：** rAF 循环 + `performance.now()` 对比
```typescript
private scheduleNextTick(): void {
  this.rafId = requestAnimationFrame(() => {
    const nextTickTime = this.tickCount * this.tickIntervalMs;
    const elapsed = performance.now() - this.startTime;
    if (elapsed >= nextTickTime) {
      this.fireTick();
    } else {
      this.scheduleNextTick(); // 还没到时间，继续等
    }
  });
}
```

### 3.4 Tick 事件处理

每个 tick 依次执行，三个音频层独立触发互不干扰：

```
fireTick()
  ├─ 预备拍阶段 (tickCount < ticksPerBar)：
  │    └─ 节拍器 ON → 播放 click（强拍/弱拍）
  │
  └─ 正常播放阶段 (tickCount >= ticksPerBar)：
       ├─ 在 bar 边界检查并应用 pending 参数变更（BPM/拍号/pattern）
       ├─ 和弦切换判断（仅在 slot===0 && beat===0 且过了第一小节后）：
       │    ├─ chordMode === 'pattern' → 每 barCount 小节换和弦
       │    └─ chordMode === 'bar' → 每 1 小节换和弦
       ├─ slot === 0 → 节拍器 ON → playClick(isStrongBeat, vol)
       ├─ 节奏型 ON && slots[slot] !== null → 播放扫弦
       │    ├─ 和弦 ON → playStrumFromEngine(direction, holdDurationMs)
       │    └─ 和弦 OFF → playMutedStrumFromEngine()
       └─ currentSlot++, 回卷 currentBeat
```

### 3.5 启动/停止

```typescript
start(): void {
  this.stop();
  this.startTime = performance.now();
  this.tickCount = 0;
  this.currentBeat = 0;
  this.currentSlot = 0;
  this.currentChordIndex = 0;
  this.scheduleNextTick();
}

stop(): void {
  if (this.tickTimerId) clearTimeout(this.tickTimerId);
  // 停止所有活跃的扫弦节点
  this.strumAudio.stopAll();
}
```

## 4. 音频层改动 (`src/audio.ts`)

### 4.0 spessasynth_lib 初始化

使用 spessasynth_lib 的 `WorkletSynthesizer` 加载完整版 SoundFont：

```typescript
import { WorkletSynthesizer } from 'spessasynth_lib';

let guitarSynth: WorkletSynthesizer | null = null;

async function initGuitarSampler(): Promise<void> {
  const ctx = this.getContext();                  // 复用 V2.0 的 AudioContext
  const basePath = import.meta.env?.BASE_URL || '/';

  // 1. 加载本地 AudioWorklet processor
  await ctx.audioWorklet.addModule(`${basePath}worklet/spessasynth_processor.min.js`);

  // 2. 加载 SF2 音色文件（92MB）
  const sf2Data = await fetch(`${basePath}samples/Acoustic_Guitars_JNv2.4.sf2`)
    .then(r => r.arrayBuffer());

  // 3. 创建 WorkletSynthesizer
  const ws = new WorkletSynthesizer(ctx);
  ws.connect(ctx.destination);

  // 4. 等待 worklet 就绪后再加载音色库
  await ws.isReady;
  await ws.soundBankManager.addSoundBank(sf2Data, 'guitar');

  // 5. 选择吉他预设（第一个预设）
  ws.programChange(0, 0);

  this.guitarSynth = ws;
  this.guitarReady = true;
}
```

**初始化时机**：在用户第一次与节奏面板交互时触发（点击播放/展开面板），不阻塞应用启动。加载期间 playClick 仍然可用。完整加载约 3-5 秒。

**fallback 机制**：`initGuitarSampler()` 失败时 `guitarReady` 保持 `false`，所有扫弦自动回退到 Web Audio 三角波合成。

### 4.1 节拍器音色 (`playClick`)

使用纯正弦波 click 声，不经过 spessasynth_lib：

```typescript
playClick(isStrongBeat: boolean, volumeMultiplier = 1.0): void {
  const audioCtx = this.getContext();
  const now = audioCtx.currentTime;

  const freq = isStrongBeat ? 1000 : 700;
  const vol = (isStrongBeat ? 0.3 : 0.18) * volumeMultiplier;

  const osc = audioCtx.createOscillator();
  osc.type = 'sine';
  osc.frequency.setValueAtTime(freq, now);

  const gain = audioCtx.createGain();
  gain.gain.setValueAtTime(vol, now);
  gain.gain.exponentialRampToValueAtTime(0.001, now + 0.06);

  osc.connect(gain).connect(audioCtx.destination);
  osc.start(now);
  osc.stop(now + 0.08);
}
```

**模式自适应音量：**
- 纯节拍器模式（和弦 OFF）：`volumeMultiplier = 1.3`
- 混合模式（和弦 ON）：`volumeMultiplier = 0.8`

### 4.2 扫弦音色 (`playStrumFromEngine`) — SF2 真人采样 + fallback

SF2 就绪时使用真人采样，否则 fallback 到三角波合成：

```typescript
playStrumFromEngine(direction: StrumDirection, holdDurationMs: number): void {
  if (this.guitarReady && this.guitarSynth) {
    this.playStrumSF2(this.engineMidiNotes, direction, holdDurationMs);
  } else {
    this.playStrum(this.engineMidiNotes, direction, holdDurationMs);  // fallback
  }
}
```

SF2 扫弦（`playStrumSF2`）：
- 下行：6 弦从低→高排序，12ms/弦，velocity=80
- 上行：仅高音 3 弦，8ms/弦，velocity=56
- NO per-strum noteOff：strums within a chord ring naturally（和弦内多次扫弦音符自然叠加）
- 仅在和弦切换时释放所有跟踪的 note（通过 `setEngineMidiNotes()` 清理）

Fallback 扫弦（`playStrum`）：
- 使用双 sawtooth 振荡器 + pick noise 瞬态
- 通过共享的 guitar 滤波器链（highpass 75Hz + peaking 220Hz + peaking 2800Hz）

### 4.3 闷音扫弦 (`playMutedStrumFromEngine`)

使用固定 MIDI 音符 + 低 velocity 触发 SF2：

```typescript
playMutedStrumFromEngine(): void {
  if (this.guitarReady && this.guitarSynth) {
    this.playMutedStrumSF2();  // 固定音符 [48,52,55,60,64,67], velocity=30
  } else {
    this.playMutedStrum();     // fallback: 白噪声 burst + lowpass filter
  }
}
```

### 4.4 和弦播放 (`playChordSF2`) — V2.1 也使用 SF2

```typescript
playChordSF2(midiNotes: number[]): void {
  if (this.guitarReady && this.guitarSynth) {
    // 释放上一个和弦的音符
    this.chordHeldNotes.forEach(n => this.guitarSynth!.noteOff(0, n));
    midiNotes.forEach(n => {
      this.guitarSynth!.noteOn(0, n, 75);
      this.chordHeldNotes.push(n);
    });
  } else {
    this.playChord(midiNotes, 2.0, true);  // fallback: 三角波
  }
}
```

### 4.5 三个音频层的独立性

| 层 | 音源 | 依赖 | 说明 |
|----|------|------|------|
| 节拍器 | 正弦波 OscillatorNode（纯 Web Audio） | 无 | 不依赖 SF2，即时可用 |
| 扫弦 | spessasynth_lib + SF2 | SF2 加载完成 | 按 pattern 格子触发，SF2 未就绪则 fallback |
| 和弦 | spessasynth_lib + SF2 | SF2 加载完成 | 单次点击用 playChordSF2()，engine 驱动用 setEngineMidiNotes() |

三者独立触发：每个 tick 单独判断各开关状态，互不影响。


## 11. 关键实现细节

### 11.1 Engine pending config 机制

`updateConfig()` 区分两种参数：开关类（metronomeOn/chordsOn/rhythmOn）立即生效；时序类（BPM/拍号/pattern）延迟到下一 bar 边界生效。

```typescript
updateConfig(config: Partial<EngineConfig>): void {
  if (!this.isRunningInternal) {
    // 未播放：直接应用
    if (config.bpm !== undefined) this.bpm = config.bpm;
    // ...
  } else {
    // 播放中：开关立即，时序排队
    if (config.metronomeOn !== undefined) this.metronomeOn = config.metronomeOn;
    if (config.bpm !== undefined) this.pendingBpm = config.bpm;
    if (config.pattern !== undefined) this.pendingPattern = config.pattern;
  }
}
```

在 `fireTick()` 正常播放阶段的 bar 边界 (`currentBeat===0 && currentSlot===0`) 检查并应用 pending 值，然后 `recalcTickInterval()`。

### 11.2 SF2 音符跟踪与止音 (Choke) 机制

所有 SF2 扫弦音符**不做 per-strum noteOff**，让同一和弦内的多次扫弦自然叠加延音。清理发生在两个时机：

1. **和弦切换时** (`setEngineMidiNotes()`)：遍历 `allStrumNotes` Set，逐一 `noteOff(0, n)`，然后清空 `strumHeldNotes` 数组
2. **停止时** (`stopAll()`)：合并 `strumHeldNotes` + `mutedHeldNotes` + `chordHeldNotes`，去重后逐一 `noteOff`

```typescript
setEngineMidiNotes(notes: number[]): void {
  this.allStrumNotes.forEach(n => this.guitarSynth?.noteOff(0, n));
  this.allStrumNotes.clear();
  this.strumHeldNotes = [];
  this.engineMidiNotes = notes;
}
```

这保证了 SF2 复音数不会被无限累积的 noteOn 耗尽。

### 11.3 Engine 和弦切换触发时机

和弦切换**不是**在最后一格触发，而是在新 bar 的**第一拍第一格** (`slot===0 && beat===0`) 触发。有前置条件：

```typescript
if (chordsOn && currentSlot===0 && currentBeat===0 && tickCount >= ticksPerBar * 2) {
  const chordIntervalBars = chordMode==='pattern' ? barCount : 1;
  const barsSincePlayback = Math.floor(tickCount / ticksPerBar) - 1; // 减去预备拍
  if (barsSincePlayback > 0 && barsSincePlayback % chordIntervalBars === 0) {
    currentChordIndex++;
    // 直接从 engine 内部设置 MIDI notes，不依赖 React 回调时序
    const idx = currentChordIndex % chordMidiNotesList.length;
    setCurrentMidiNotes(chordMidiNotesList[idx]);
    onChordChange?.(currentChordIndex);
  }
}
```

关键设计：`tickCount >= ticksPerBar * 2` 确保至少过了预备拍（1 bar）+ 第一个正式 bar（1 bar）后才开始切换。和弦 MIDI notes 由 engine 直接推送到 audio 层，不经过 React state 回调以避免异步时序问题。

### 11.4 onTick 回调的节流策略

Engine 每个 tick（约 150ms@100BPM）都触发 `onTick`，但 React state 更新被节流：

```typescript
let prevTickSlot = -1;
engine.onTick = (info) => {
  if (info.slot === 0 || info.slot < prevTickSlot) {
    // 仅在 beat 边界 (slot===0) 或回卷时更新 React state
    setCurrentTickBeat(info.beat);
    setCurrentTickSlot(info.slot);
  }
  prevTickSlot = info.slot;
};
```

这确保预览网格高亮在非边界格子时仍能通过 animation 推进（因为 `currentTickBeat` 在 slot===0 时更新，触发 re-render），但将 React re-render 频率从 16分音符级（~150ms）降到拍级（~600ms@100BPM）。

### 11.5 6/8 拍 BPM 换算

6/8 的 BPM 以附点四分音符（dotted quarter）为单位，每个附点四分音符 = 3 个八分音符：

```typescript
const bpmFactor = this.timeSignature === '6/8' ? 3 : 1;
this.tickIntervalMs = (60000 / (this.bpm * bpmFactor)) / subdivision;
```

所以 `6/8`，`subdivision=3`，`BPM=100` 时：`interval = 60000 / (100*3) / 3 = 66.7ms`（每个八分音符 66.7ms）。

```
User clicks Play
  └─ handlePlay()
       ├─ triggerGuitarInit()     → synth.initGuitarSampler() [首次触发]
       ├─ synth.setEngineMidiNotes(chords[0].midiNotes)  → 设置当前和弦
       ├─ engine.setChordMidiNotesList([...all chords...])
       ├─ engine.configure({...})   → 写 BPM/拍号/pattern/开关/chordMode
       ├─ engine.onTick = (...)     → 注册回调（节流 React state 更新）
       ├─ engine.onChordChange = (...) → 注册回调（更新 activeChordIndex）
       └─ engine.start()
            └─ scheduleNextTick()   → rAF 循环
                 └─ fireTick()
                      ├─ 预备拍: playClick()
                      └─ 正常播放:
                           ├─ bar边界: 应用pending配置
                           ├─ bar边界: 和弦切换 → setEngineMidiNotes()
                           │            → engine 直接推送 MIDI 到 audio 层
                           ├─ slot===0: playClick()
                           └─ strum slot: playStrumFromEngine(dir, dur)
                                ├─ SF2就绪 → playStrumSF2()
                                │              ├─ noteOn(0, note, vel) × 6弦
                                │              └─ 跟踪到 strumHeldNotes + allStrumNotes
                                └─ SF2未就绪 → playStrum() (fallback)
                                                 ├─ 双 sawtooth + pick noise
                                                 └─ 吉他滤波器链 → destination

User clicks Stop
  └─ handleStop()
       ├─ engine.stop()   → 清 rAF, 清 timer
       ├─ synth.stopAll() → 清 setInterval + 所有 OscillatorNode
       │                    + releaseAllHeldNotes() → 逐一 noteOff SF2
       └─ setState × 4
```
